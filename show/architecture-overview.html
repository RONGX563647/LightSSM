关键设计点

三级缓存

解决循环依赖问题，通过提前暴露早期引用打破依赖环

BeanPostProcessor

提供扩展点，在 Bean 初始化前后进行自定义处理，是 AOP 的基础

延迟实例化

Bean 只在第一次 getBean() 时创建，避免不必要的资源消耗

四、核心源码解析 - DefaultListableBeanFactory

1. 数据结构定义

public class DefaultListableBeanFactory implements ListableBeanFactory { // BeanDefinition 注册表 private final Map<String, BeanDefinition> beanDefinitionMap = new ConcurrentHashMap<>(256); // 一级缓存：完整 Bean 实例 private final Map<String, Object> singletonObjects = new ConcurrentHashMap<>(256); // 二级缓存：早期 Bean 引用（未完成属性填充） private final Map<String, Object> earlySingletonObjects = new ConcurrentHashMap<>(16); // 三级缓存：Bean 工厂（用于创建早期引用） private final Map<String, ObjectFactory<?>> singletonFactories = new ConcurrentHashMap<>(16); // 记录正在创建的 Bean 名称 private final Set<String> singletonsCurrentlyInCreation = Collections.newSetFromMap(new ConcurrentHashMap<>(16)); // BeanPostProcessor 集合 private final List<BeanPostProcessor> beanPostProcessors = new ArrayList<>(); }

2. 核心方法 doGetBean()

protected <T> T doGetBean(String name, Class<T> requiredType, Object[] args) { BeanDefinition bd = this.beanDefinitionMap.get(name); if (bd == null) { throw new NoSuchBeanDefinitionException(name); } if (bd.isSingleton()) { // 尝试从缓存获取 Object sharedInstance = getSingleton(name); if (sharedInstance == null) { sharedInstance = createSingletonBean(name, bd); } return (T) sharedInstance; } else { // 原型 Bean 每次创建新实例 return (T) createBean(name, bd); } }

3. Bean 创建流程 doCreateBean()

protected Object doCreateBean(String beanName, BeanDefinition bd) { // 1. 实例化 Bean Object beanInstance = instantiateBean(beanName, bd); // 2. 提前暴露引用（放入三级缓存） boolean earlySingletonExposure = bd.isSingleton() && this.singletonsCurrentlyInCreation.contains(beanName); if (earlySingletonExposure) { addSingletonFactory(beanName, () -> getEarlyBeanReference(beanName, bd, beanInstance)); } // 3. 属性填充（依赖注入） populateBean(beanName, bd, beanInstance); // 4. 初始化 Bean Object exposedObject = initializeBean(beanName, beanInstance, bd); // 5. 处理早期引用冲突 if (earlySingletonExposure) { Object earlySingletonReference = getSingleton(beanName); if (earlySingletonReference != null) { exposedObject = earlySingletonReference; } } return exposedObject; }

为什么需要三级缓存？

一级缓存 singletonObjects：存储完整的、初始化完成的 Bean 实例。这是最终目标，所有单例 Bean 最终都会存放在这里。

二级缓存 earlySingletonObjects：存储早期的 Bean 引用（已实例化但未完成属性填充）。用于解决循环依赖，让其他 Bean 可以提前获取到引用。

三级缓存 singletonFactories：存储 ObjectFactory 工厂对象。通过工厂可以按需创建早期引用，延迟了早期引用的创建时机，是解决 AOP 代理问题的关键。

核心机制

提前暴露：BeanA 在实例化后、属性填充前，就将工厂对象放入三级缓存，这样其他 Bean 就可以通过工厂获取到 BeanA 的早期引用。

早期引用：从三级缓存的工厂创建早期引用，放入二级缓存。早期引用只是实例化后的对象，还未完成属性填充和初始化。

引用传递：当 BeanB 完成创建后放入一级缓存，BeanA 继续创建时注入的 BeanB 已经是完整实例，从而打破循环依赖。
