09 - 插件系统与扩展机制

1. 插件架构概述

1.1 插件的作用

MyBatis的插件系统允许开发者在不修改核心代码的前提下，扩展或修改框架行为。

应用场景：

  • SQL性能监控

  • 分页插件

  • 多租户数据源隔离

  • 数据加密解密

  • 审计日志


1.2 可拦截对象

MyBatis允许拦截四大对象：

Executor（执行器）
├── update
├── query
├── commit
└── rollback

StatementHandler（语句处理器）
├── prepare
├── parameterize
├── batch
└── query


ParameterHandler（参数处理器）
├── setParameters
└── getParameters


ResultSetHandler（结果集处理器）
├── handleResultSets
└── handleOutputParameters

2. 插件接口设计

2.1 Interceptor接口

LightSSM实现：

public interface Interceptor {
    Object intercept(Invocation invocation) throws Throwable;
    Object plugin(Object target);
    void setProperties(Properties properties);
}

MyBatis官方Interceptor接口：

public interface Interceptor {
    Object intercept(Invocation invocation) throws Throwable;
    Object plugin(Object target);
    void setProperties(Properties properties);
}

接口完全一致。

2.2 Intercepts注解

LightSSM实现：

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface Intercepts {
    Signature[] value();
}

Signature注解：

@Retention(RetentionPolicy.RUNTIME)
@Target({})
public @interface Signature {
    Class type();      // 拦截的类
    String method();      // 拦截的方法
    Class[] args();    // 方法参数
}

2.3 使用示例

@Intercepts({
    @Signature(
        type = Executor.class,
        method = "query",
        args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class}
    )
})
public class SqlMonitorInterceptor implements Interceptor {
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        long startTime = System.currentTimeMillis();
        
        // 执行目标方法
        Object result = invocation.proceed();
        
        long endTime = System.currentTimeMillis();
        System.out.println("SQL执行时间：" + (endTime - startTime) + "ms");
        
        return result;
    }
    
    @Override
    public Object plugin(Object target) {
        return Plugin.wrap(target, this);
    }
    
    @Override
    public void setProperties(Properties properties) {
        // 设置插件属性
    }
}

3. 插件责任链

3.1 InterceptorChain

LightSSM实现：

public class InterceptorChain {
    private final List interceptors = new ArrayList<>();
    
    public void addInterceptor(Interceptor interceptor) {
        interceptors.add(interceptor);
    }
    
    public Object pluginAll(Object target) {
        Object wrapped = target;
        // 逆序包装，形成责任链
        for (int i = interceptors.size() - 1; i >= 0; i--) {
            wrapped = interceptors.get(i).plugin(wrapped);
        }
        return wrapped;
    }
    
    public List getInterceptors() {
        return Collections.unmodifiableList(interceptors);
    }
}

3.2 与MyBatis官方对比

MyBatis官方的InterceptorChain：

public Object pluginAll(Object target) {
    Object wrapped = target;
    for (Interceptor interceptor : interceptors) {
        wrapped = interceptor.plugin(wrapped);
    }
    return wrapped;
}

对比分析：

| 功能 | MyBatis官方 | LightSSM |
|-----|-----------|----------|
| 插件顺序 | 正序包装 | 逆序包装 |
| 包装逻辑 | 相同 | 相同 |
| 不可变列表 | 不支持 | 支持getInterceptors返回不可变列表 |

4. 插件代理机制

4.1 Plugin类

LightSSM实现：

public class Plugin implements InvocationHandler {
    private final Object target;
    private final Interceptor interceptor;
    private final Map, Set> signatureMap;
    
    private Plugin(Object target, Interceptor interceptor, 
                   Map, Set> signatureMap) {
        this.target = target;
        this.interceptor = interceptor;
        this.signatureMap = signatureMap;
    }
    
    public static Object wrap(Object target, InterceptorChain interceptorChain) {
        List interceptors = interceptorChain.getInterceptors();
        for (Interceptor interceptor : interceptors) {
            target = wrap(target, interceptor);
        }
        return target;
    }
    
    public static Object wrap(Object target, Interceptor interceptor) {
        Map, Set> signatureMap = getSignatureMap(interceptor);
        Class type = target.getClass();
        Class[] interfaces = getAllInterfaces(type);
        
        // 检查是否需要代理
        if (canWrapSignature(interfaces, signatureMap)) {
            return Proxy.newProxyInstance(
                type.getClassLoader(),
                interfaces,
                new Plugin(target, interceptor, signatureMap));
        }
        return target;
    }
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        Set methods = signatureMap.get(method.getDeclaringClass());
        
        if (methods != null && methods.contains(method)) {
            // 需要拦截，创建Invocation
            return interceptor.intercept(
                new Invocation(target, method, args));
        }
        
        // 不需要拦截，直接执行
        return method.invoke(target, args);
    }
    
    private static Map, Set> getSignatureMap(Interceptor interceptor) {
        Intercepts interceptsAnnotation = interceptor.getClass().getAnnotation(Intercepts.class);
        if (interceptsAnnotation == null) {
            throw new PluginException("No @Intercepts annotation found");
        }
        
        Signature[] sigs = interceptsAnnotation.value();
        Map, Set> signatureMap = new HashMap<>();
        
        for (Signature sig : sigs) {
            Set methods = signatureMap.computeIfAbsent(sig.type(), k -> new HashSet<>());
            try {
                Method method = sig.type().getMethod(sig.method(), sig.args());
                methods.add(method);
            } catch (NoSuchMethodException e) {
                throw new PluginException("Could not find method on " + sig.type() 
                    + " named " + sig.method(), e);
            }
        }
        
        return signatureMap;
    }
}

4.2 Invocation类

LightSSM实现：

public class Invocation {
    private final Object target;
    private final Method method;
    private final Object[] args;
    
    public Invocation(Object target, Method method, Object[] args) {
        this.target = target;
        this.method = method;
        this.args = args;
    }
    
    public Object getTarget() {
        return target;
    }
    
    public Method getMethod() {
        return method;
    }
    
    public Object[] getArgs() {
        return args;
    }
    
    public Object proceed() throws InvocationTargetException, IllegalAccessException {
        method.setAccessible(true);
        return method.invoke(target, args);
    }
}

5. 插件实现示例

5.1 SQL性能监控插件

@Intercepts({
    @Signature(
        type = Executor.class,
        method = "query",
        args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class}
    ),
    @Signature(
        type = Executor.class,
        method = "update",
        args = {MappedStatement.class, Object.class}
    )
})
public class PerformanceInterceptor implements Interceptor {
    
    private long maxTime = 1000; // 默认最大执行时间1秒
    private boolean throwException = false;
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        long startTime = System.currentTimeMillis();
        
        // 获取SQL信息
        MappedStatement ms = (MappedStatement) invocation.getArgs()[0];
        String sqlId = ms.getId();
        
        try {
            Object result = invocation.proceed();
            
            long endTime = System.currentTimeMillis();
            long executionTime = endTime - startTime;
            
            if (executionTime > maxTime) {
                String message = String.format("SQL执行超时 [%s]: %dms", sqlId, executionTime);
                if (throwException) {
                    throw new RuntimeException(message);
                } else {
                    System.err.println(message);
                }
            }
            
            return result;
        } catch (Exception e) {
            throw e;
        }
    }
    
    @Override
    public Object plugin(Object target) {
        return Plugin.wrap(target, this);
    }
    
    @Override
    public void setProperties(Properties properties) {
        String maxTimeStr = properties.getProperty("maxTime");
        if (maxTimeStr != null) {
            maxTime = Long.parseLong(maxTimeStr);
        }
        
        String throwExceptionStr = properties.getProperty("throwException");
        if (throwExceptionStr != null) {
            throwException = Boolean.parseBoolean(throwExceptionStr);
        }
    }
}

6. 插件在Configuration中的集成

6.1 插件注册

LightSSM实现：

public class Configuration {
    protected final InterceptorChain interceptorChain = new InterceptorChain();
    
    public void addInterceptor(Interceptor interceptorInstance) {
        interceptorChain.addInterceptor(interceptorInstance);
    }
    
    public StatementHandler newStatementHandler(...) {
        StatementHandler statementHandler = new PreparedStatementHandler(...);
        // 应用插件代理
        statementHandler = (StatementHandler) interceptorChain.pluginAll(statementHandler);
        return statementHandler;
    }
    
    public ParameterHandler newParameterHandler(...) {
        ParameterHandler parameterHandler = mappedStatement.getLang()
            .createParameterHandler(mappedStatement, parameterObject, boundSql);
        // 应用插件代理
        parameterHandler = (ParameterHandler) interceptorChain.pluginAll(parameterHandler);
        return parameterHandler;
    }
    
    public ResultSetHandler newResultSetHandler(...) {
        ResultSetHandler resultSetHandler = new DefaultResultSetHandler(...);
        // 应用插件代理
        resultSetHandler = (ResultSetHandler) interceptorChain.pluginAll(resultSetHandler);
        return resultSetHandler;
    }
    
    public Executor newExecutor(Transaction transaction) {
        Executor executor = new SimpleExecutor(this, transaction);
        // 应用插件代理
        executor = (Executor) interceptorChain.pluginAll(executor);
        return executor;
    }
}

8. 面试考点

8.1 高频问题

1. MyBatis插件原理是什么？
- 动态代理（JDK Proxy）
- 拦截四大对象
- 责任链模式

2. MyBatis可以拦截哪些对象？
- Executor
- StatementHandler
- ParameterHandler
- ResultSetHandler

3. 如何编写一个分页插件？
- 拦截Executor.query()
- 修改SQL添加LIMIT
- 修改BoundSql

8.2 深度问题

1. 插件的执行顺序？
- 按注册顺序逆序包装
- 正序执行（先注册的先执行）

2. 为什么插件要逆序包装？
- 保证执行顺序与注册顺序一致
- 类似栈结构（LIFO）

3. 插件的性能影响？
- 每次方法调用都要经过代理
- 多个插件形成责任链
- 建议控制插件数量

9. 总结

LightSSM的插件系统实现了：

  • Interceptor接口：插件标准接口
  • InterceptorChain：插件责任链
  • Plugin：动态代理实现
  • Invocation：方法调用封装
  • @Intercepts/@Signature：注解配置
通过对照MyBatis官方源码，可以看到LightSSM保留了插件系统的核心机制，非常适合学习理解AOP和插件原理。
