核心概念解析

1. 循环依赖问题

问题场景

当 BeanA 依赖 BeanB，BeanB 又依赖 BeanA 时，如何避免无限递归创建？三级缓存机制是经典解决方案。

2. 三级缓存结构

  • 一级缓存：singletonObjects - 存储完整的 Bean 实例
  • 二级缓存：earlySingletonObjects - 存储早期 Bean 引用
  • 三级缓存：singletonFactories - 存储 Bean 工厂

3. AOP 代理机制

两种代理方式

  • JDK 动态代理：基于接口实现
  • CGLIB 代理：基于子类实现

选择策略：优先使用 JDK 代理，否则使用 CGLIB。

4. 动态 SQL 解析

SqlNode 树结构

  • MixedSqlNode：混合节点
  • IfSqlNode：条件节点
  • TextSqlNode：文本节点

1. SqlNode 接口设计

// SqlNode 接口 interface SqlNode { apply(context) } // MixedSqlNode - 混合节点 class MixedSqlNode implements SqlNode { contents = List<SqlNode> apply(context) { for node in contents: node.apply(context) } } // IfSqlNode - 条件节点 class IfSqlNode implements SqlNode { test = String contents = SqlNode apply(context) { if evaluator.evaluateBoolean(test, context.getBindings()): contents.apply(context) } } // TextSqlNode - 文本节点 class TextSqlNode implements SqlNode { text = String apply(context) { parser = new GenericTokenParser("#{", "}", token -> context.getBindings().get(token)) context.appendSql(parser.parse(text)) } }

2. 插件机制

// 性能监控插件 class PerformanceInterceptor implements Interceptor { intercept(invocation) { start = System.currentTimeMillis() try { return invocation.proceed() } finally { elapsed = System.currentTimeMillis() - start logger.info("SQL 执行耗时：{}ms", elapsed) } } plugin(target) { return Plugin.wrap(target, this) } } // 插件包装逻辑 class Plugin { static wrap(target, interceptor) { signatureMap = getSignatureMap(interceptor) if shouldWrap(target.getClass(), signatureMap): return Proxy.newProxyInstance( target.getClass().getInterfaces(), new PluginInvocationHandler(target, interceptor) ) return target } }

关键点回顾

  • 循环依赖：三级缓存 + 提前暴露引用
  • AOP 代理：JDK 动态代理 + CGLIB
  • 动态 SQL：SqlNode 树 + 递归 apply()
  • 插件扩展：责任链 + 代理包装
