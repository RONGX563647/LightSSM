循环依赖与 AOP

1. 循环依赖问题

问题场景

当 BeanA 依赖 BeanB，BeanB 又依赖 BeanA 时，如何避免无限递归创建？三级缓存机制是经典解决方案。

2. 三级缓存结构

  • 一级缓存：singletonObjects - 存储完整的 Bean 实例
  • 二级缓存：earlySingletonObjects - 存储早期 Bean 引用
  • 三级缓存：singletonFactories - 存储 Bean 工厂，用于创建早期引用

3. 为什么必须三级缓存？

如果只有两级：无法处理 AOP 代理场景。当 Bean 被 AOP 增强时，二级缓存存储的是早期引用，无法延迟代理对象的创建。

三级缓存的核心价值：ObjectFactory 的 getObject() 方法中会调用 BeanPostProcessor.getEarlyBeanReference()，在这里可以应用 AOP 代理。

4. AOP 代理机制

两种代理方式

  • JDK 动态代理：基于接口实现，使用 InvocationHandler
  • CGLIB 代理：基于子类实现，使用 MethodInterceptor

选择策略：如果目标对象实现了接口，优先使用 JDK 代理；否则使用 CGLIB。

动态 SQL 与插件

1. 动态 SQL 解析

SqlNode 树结构

XML 动态 SQL 被解析为 SqlNode 树，每个标签对应一个 SqlNode 节点。

  • MixedSqlNode：混合节点，包含多个子节点
  • IfSqlNode：条件节点，根据表达式决定是否包含内容
  • TextSqlNode：文本节点，包含静态 SQL 和占位符
  • TrimSqlNode：修剪节点，处理 WHERE、SET 等前缀

2. SqlNode 接口设计

// SqlNode 接口 interface SqlNode { apply(context) } // MixedSqlNode - 混合节点 class MixedSqlNode implements SqlNode { contents = List<SqlNode> apply(context) { for node in contents: node.apply(context) } } // IfSqlNode - 条件节点 class IfSqlNode implements SqlNode { test = String contents = SqlNode apply(context) { if evaluator.evaluateBoolean(test, context.getBindings()): contents.apply(context) } } // TextSqlNode - 文本节点 class TextSqlNode implements SqlNode { text = String apply(context) { // 替换参数占位符 #{xxx} parser = new GenericTokenParser("#{", "}", token -> context.getBindings().get(token)) context.appendSql(parser.parse(text)) } }

3. 插件机制

责任链模式

插件通过责任链模式包装目标对象，在方法执行前后添加增强逻辑。常见用途：性能监控、分页、SQL 审计等。

4. 插件实现示例

// 性能监控插件 class PerformanceInterceptor implements Interceptor { intercept(invocation) { start = System.currentTimeMillis() try { return invocation.proceed() } finally { elapsed = System.currentTimeMillis() - start logger.info("SQL 执行耗时：{}ms", elapsed) } } plugin(target) { return Plugin.wrap(target, this) } } // 插件包装逻辑 class Plugin { static wrap(target, interceptor) { signatureMap = getSignatureMap(interceptor) if shouldWrap(target.getClass(), signatureMap): return Proxy.newProxyInstance( target.getClass().getInterfaces(), new PluginInvocationHandler(target, interceptor) ) return target } }

5. 核心难点总结

关键点回顾

  • 循环依赖：三级缓存 + 提前暴露引用
  • AOP 代理：JDK 动态代理 + CGLIB，方法调用链
  • 动态 SQL：SqlNode 树 + 递归 apply()
  • 插件扩展：责任链 + 代理包装
