MVC 组件职责

DispatcherServlet

职责：前端控制器，统一接收所有 HTTP 请求，协调各组件完成请求处理。

类比：类似于公司的前台，所有来访者都需要先经过前台登记和引导。

HandlerMapping

职责：根据 URL 查找对应的 Controller 方法，返回 HandlerExecutionChain。

实现：维护 URL 到 HandlerMethod 的映射表，支持模糊匹配。

HandlerAdapter

职责：统一调用 Handler 方法，屏蔽不同 Handler 的调用差异。

为什么需要：Controller 可能有多种实现方式（注解、接口等），需要适配器统一调用。

请求处理 9 步曲

  • 1. DispatcherServlet 接收请求
  • 2. HandlerMapping 查找处理器
  • 3. 获取 HandlerAdapter 适配器
  • 4. 执行拦截器 preHandle()
  • 5. HandlerAdapter 执行 Controller
  • 6. 执行业务逻辑，返回结果
  • 7. 执行拦截器 postHandle()
  • 8. ViewResolver 解析视图
  • 9. 执行拦截器 afterCompletion()

LightSSM 源码实现

1. DispatcherServlet 核心

// 前端控制器：统一处理请求 service(request, response) { try { // 1. 获取 Handler mappedHandler = getHandler(request) if mappedHandler == null: response.sendError(404) return // 2. 获取 HandlerAdapter ha = getHandlerAdapter(mappedHandler.handler) // 3. 执行拦截器前置处理 if !mappedHandler.applyPreHandle(request, response): return // 4. 执行 Handler 方法 mv = ha.handle(request, response, mappedHandler.handler) // 5. 执行拦截器后置处理 mappedHandler.applyPostHandle(request, response, mv) // 6. 渲染视图 render(mv, request, response) } catch (Exception e): // 异常处理 } finally { // 7. 执行拦截器完成处理 triggerAfterCompletion() } }

2. HandlerMapping 映射表

// URL 映射表构建与查找 class RequestMappingHandlerMapping { handlerMap = Map<String, HandlerMethod> registerHandlerMethod(bean, method, mapping) { urlPattern = mapping.value() handlerMethod = new HandlerMethod(bean, method) handlerMap.put(urlPattern, handlerMethod) } getHandler(request) { url = request.getRequestURI() for pattern in handlerMap.keySet(): if matchUrl(pattern, url): return new HandlerExecutionChain( handlerMap.get(pattern) ) return null } }

3. HandlerAdapter 调用

// 适配器：统一调用 Handler class RequestMappingHandlerAdapter { handle(request, response, handler) { handlerMethod = (HandlerMethod) handler // 1. 解析方法参数 args = getMethodArguments(handlerMethod, request, response) // 2. 反射调用方法 returnValue = handlerMethod.invoke(args) // 3. 处理返回值 if handlerMethod.hasAnnotation(ResponseBody): writeWithMessageConverter(returnValue, response) return null else: return new ModelAndView( handlerMethod.getViewName(), returnValue ) } }

4. 参数解析机制

// 方法参数解析 getMethodArguments(handlerMethod, request, response) { params = method.getParameters() args = new Object[params.length] for i in 0..params.length: param = params[i] if param.hasAnnotation(PathVariable): pv = param.getAnnotation(PathVariable) args[i] = request.getAttribute(pv.value()) else if param.hasAnnotation(RequestParam): rp = param.getAnnotation(RequestParam) args[i] = request.getParameter(rp.value()) else if param.type == HttpServletRequest: args[i] = request else if param.type == HttpServletResponse: args[i] = response return args }

2. 源码对比：请求分发核心

原生 Spring - DispatcherServlet.doDispatch()
protected void doDispatch( HttpServletRequest request, HttpServletResponse response) { // 1. 查找 Handler HandlerExecutionChain mappedHandler = getHandler(processedRequest); // 2. 查找 HandlerAdapter HandlerAdapter ha = getHandlerAdapter(mappedHandler); // 3. 执行拦截器前置处理 if (!mappedHandler .applyPreHandle(...)) { return; } // 4. 执行 Controller 方法 ModelAndView mv = ha.handle( processedRequest, response, mappedHandler.getHandler()); // 5. 执行拦截器后置处理 mappedHandler.applyPostHandle(...); // 6. 渲染视图 processDispatchResult(...); }
LightSSM - DispatcherServlet.service()
protected void service( HttpServletRequest request, HttpServletResponse response) { // 1. 获取请求路径 String url = request .getRequestURI(); // 2. 查找 Handler Handler handler = handlerMap.get(url); if (handler == null) { response.sendError(404); return; } // 3. 参数绑定 Object[] args = bindParams( request, handler); // 4. 反射执行 Object result = handler.getMethod() .invoke(handler.getBean(), args); // 5. 处理响应 handleResponse(response, result); }

💡 架构设计洞察

原生 Spring 的强大在于扩展性：通过 HandlerMapping、HandlerAdapter 的接口抽象，支持多种请求映射方式和处理器类型。而 LightSSM 的精髓在于理解本质：MVC 的核心就是"URL → Controller 方法 → 响应"的映射过程。通过手写简化版，你能清晰看到每个环节的作用。

🎯 面试核心：MVC 的本质

  • MVC 的核心：Model（数据）+ View（视图）+ Controller（控制器）的分离
  • DispatcherServlet 的作用：前端控制器，统一入口，协调各个组件
  • HandlerMapping 的作用：URL 到处理器的映射关系
  • HandlerAdapter 的作用：适配器模式，支持多种处理器类型
  • 参数绑定的本质：将 HTTP 请求参数转换为 Java 方法参数
