Spring 设计理念

核心接口层次

BeanFactory：最底层的 Bean 工厂接口，提供最基本的 Bean 获取功能。

ListableBeanFactory：扩展接口，提供按类型查找 Bean 的能力。

ApplicationContext：更高级的接口，提供国际化、事件传播等企业级功能。

LightSSM vs Spring

  • 相同点：都采用 BeanFactory 作为核心接口，使用三级缓存解决循环依赖
  • 简化点：LightSSM 将 ApplicationContext 和 BeanFactory 合并，去除了企业级功能
  • 实现差异：Spring 使用 XmlBeanDefinitionReader 解析 XML，LightSSM 直接扫描注解

// 数据结构定义 class DefaultListableBeanFactory { // BeanDefinition 注册表 private Map<String, BeanDefinition> beanDefinitionMap // 一级缓存：完整 Bean 实例 private Map<String, Object> singletonObjects // 二级缓存：早期 Bean 引用 private Map<String, Object> earlySingletonObjects // 三级缓存：Bean 工厂 private Map<String, ObjectFactory> singletonFactories // 记录正在创建的 Bean private Set<String> singletonsCurrentlyInCreation // BeanPostProcessor 集合 private List<BeanPostProcessor> beanPostProcessors }

2. 核心方法 doGetBean()

// 核心方法：获取 Bean doGetBean(name, requiredType) { bd = beanDefinitionMap.get(name) if bd == null: throw NoSuchBeanDefinitionException(name) if bd.isSingleton(): sharedInstance = getSingleton(name) if sharedInstance == null: sharedInstance = createSingletonBean(name, bd) return sharedInstance else: return createBean(name, bd) }

3. 三级缓存 getSingleton()

// 三级缓存查找逻辑 getSingleton(beanName) { // 第一级：查找完整 Bean singletonObject = singletonObjects.get(beanName) if singletonObject == null && isSingletonCurrentlyInCreation(beanName): // 第二级：查找早期引用 singletonObject = earlySingletonObjects.get(beanName) if singletonObject == null: // 第三级：通过工厂创建早期引用 factory = singletonFactories.get(beanName) if factory != null: singletonObject = factory.getObject() earlySingletonObjects.put(beanName, singletonObject) singletonFactories.remove(beanName) return singletonObject }

4. Bean 创建 doCreateBean()

// Bean 创建流程 doCreateBean(beanName, bd) { // 1. 实例化 Bean beanInstance = instantiateBean(beanName, bd) // 2. 提前暴露引用（放入三级缓存） if bd.isSingleton() && singletonsCurrentlyInCreation.contains(beanName): addSingletonFactory(beanName, () -> getEarlyBeanReference(beanName, beanInstance)) // 3. 属性填充（依赖注入） populateBean(beanName, bd, beanInstance) // 4. 初始化 Bean exposedObject = initializeBean(beanName, beanInstance, bd) // 5. 处理早期引用冲突 if earlySingletonExposure: earlyRef = getSingleton(beanName) if earlyRef != null: exposedObject = earlyRef return exposedObject }

2. 源码逐行对比：getSingleton()

原生 Spring 源码
// AbstractAutowireCapableBeanFactory protected Object getSingleton(String beanName) { // 1. 从一级缓存查找 Object singletonObject = this .singletonObjects.get(beanName); // 2. 一级缓存未命中且正在创建 if (singletonObject == null && isSingletonCurrentlyInCreation(beanName)) { // 3. 从二级缓存查找 singletonObject = this .earlySingletonObjects.get(beanName); // 4. 二级缓存未命中，从三级缓存获取 if (singletonObject == null) { ObjectFactory<?> factory = this .singletonFactories.get(beanName); if (factory != null) { // 5. 通过工厂创建早期引用 singletonObject = factory.getObject(); // 6. 放入二级缓存 this.earlySingletonObjects .put(beanName, singletonObject); // 7. 移除三级缓存 this.singletonFactories .remove(beanName); } } } return singletonObject; }

💡 核心发现

LightSSM 的 getSingleton() 实现与原生 Spring 几乎完全一致！这证明了 Spring 三级缓存设计的精妙之处。核心逻辑完全相同：一级缓存 → 二级缓存 → 三级缓存工厂，层层递进，确保循环依赖能够正确解决。

🎯 面试核心：三级缓存的本质

  • 一级缓存是目标：存储完整可用的 Bean 实例，这是最终目标
  • 二级缓存是过渡：存储早期引用，解决"我依赖你，你依赖我"的死锁问题
  • 三级缓存是延迟：通过 ObjectFactory 延迟创建早期引用，给 AOP 留出织入机会
  • 核心思想：提前暴露、延迟创建、缓存复用、分层管理
  • 为什么需要三级？如果只有两级，无法支持 AOP 代理的提前织入。三级缓存通过工厂模式，让 AOP 可以在 Bean 完全初始化之前就创建代理对象。

📝 详细步骤说明

  1. 创建 A：实例化 A，将 A 的 ObjectFactory 放入三级缓存
  2. A 依赖 B：A 进行属性注入时发现需要 B
  3. 创建 B：实例化 B，将 B 的 ObjectFactory 放入三级缓存
  4. B 依赖 A：B 进行属性注入时发现需要 A，触发循环依赖
  5. 从三级缓存获取 A：找到 A 的 ObjectFactory，调用 getObject() 创建 A 的早期引用
  6. 转移缓存：将 A 的早期引用放入二级缓存，移除三级缓存
  7. B 注入 A：B 成功注入 A 的早期引用，B 完成创建
  8. A 注入 B：A 注入已完成的 B，A 完成创建
