## 模块概述

基础工具模块是整个 ORM 框架的地基，提供了最底层的支持和基础设施。本模块包含五大子模块：

1. **annotations** - 注解定义
2. **io** - 资源加载
3. **parsing** - SQL解析
4. **transaction** - 事务管理
5. **type** - 类型处理

## 1. 注解模块 (annotations)

### 1.1 核心注解

本框架定义了4个SQL注解，用于标记DAO方法对应的SQL语句：

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface Select {
    String[] value();
}
```

#### 注解设计分析

| 属性 | 说明 |
|------|------|
| `@Retention(RUNTIME)` | 运行时保留，通过反射可获取注解信息 |
| `@Target(METHOD)` | 作用在方法上 |
| `String[] value()` | 支持多个SQL语句（用于返回List结果） |

**为什么用注解？**
- 相比XML，注解更简洁，适合简单查询
- 注解定义在方法上，代码与SQL同处一处，便于维护
- 支持多SQL实现分页查询等复杂场景

### 1.2 其他注解

| 注解 | 用途 |
|------|------|
| `@Insert` | 插入语句 |
| `@Update` | 更新语句 |
| `@Delete` | 删除语句 |

### 1.3 学习要点

**Java注解基础**：
```java
// 元注解种类
@Retention  // 保留策略
@Target    // 作用目标
@Documented // 文档输出
@Inherited // 可继承
```

## 2. 资源加载模块 (io)

### 2.1 Resources 类

负责从classpath或文件系统加载资源文件。

```java
public class Resources {
    public static InputStream getResourceAsStream(String resource);
    public static Reader getResourceAsReader(String resource);
    public static Class getResourceAsClass(String resource);
}
```

### 2.2 设计原理

**类加载器选择**：
1. 首先尝试 `Thread.currentThread().getContextClassLoader()`
2. 失败则使用 `Resources.class.getClassLoader()`

**为什么这样设计？**
- 支持Web环境下的类加载器传递
- 避免ClassLoader隔离问题

### 2.3 学习要点

```

                     ↓

```

### 3.1 GenericTokenParser - 核心解析器

```java
public class GenericTokenParser {
    private final String openToken;   // 开始标记，如 #{
    private final String closeToken; // 结束标记，如 }
    private final TokenHandler handler;
    
    public String parse(String text) {
        // 核心循环逻辑
        while (start > -1) {
            // 找到参数内容 调用 handler.handleToken 处理
        }
    }
}
```

### 3.2 底层实现原理

**解析流程**：

```
原始SQL: SELECT * FROM user WHERE id = #{id} AND name = #{name}
                     ↓
第1次循环: 处理 #{id}
第2次循环: 处理 #{name}
                     ↓
结果: SELECT * FROM user WHERE id = ? AND name = ?
```

**关键代码解析**：
```java
// 1. 找到开始标记位置
int start = text.indexOf(openToken, offset);

// 2. 找到结束标记位置  
int end = text.indexOf(closeToken, start);

// 3. 提取参数名
String content = new String(src, offset, end - offset);

// 4. 调用处理器替换
builder.append(handler.handleToken(content));
```

### 3.3 TokenHandler 接口

```java
public interface TokenHandler {
    String handleToken(String content);
}
```

**处理器实现**：
- `BindingTokenHandler` - 参数绑定
- `VariableTokenHandler` - 变量替换

### 3.4 学习要点

**#{} vs ${}**：

| 特性 | #{parameter} | ${parameter} |
|------|-------------|-------------|
| 处理方式 | 预编译?占位 | 字符串拼接 |
| SQL注入 | 安全 | 危险 |
| 性能 | 优 | 差 |

**解析算法复杂度**：O(n)，n为SQL长度

## 5. 类型处理模块 (type) - 重点

### 5.1 TypeHandler 接口

类型处理器负责Java类型与JDBC类型之间的双向转换。

```java
public interface TypeHandler<T> {
    // Java → JDBC: 设置参数到PreparedStatement
    void setParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType);
    
    // JDBC → Java: 从ResultSet读取结果
    T getResult(ResultSet rs, String columnName);
    T getResult(ResultSet rs, int columnIndex);
}
```

### 5.2 转换流程图

```
写操作 (Java → DB):
User对象 ──→ user.getId() ──→ IntegerTypeHandler ──→ ps.setInt(1, id)
                  ↓
              PreparedStatement
                  ↓
              SQL: INSERT INTO user VALUES (?)

读操作 (DB → Java):
ResultSet ──→ rs.getInt("id") ──→ IntegerTypeHandler ──→ user.setId(id)
              ↓
           ResultSet
              ↓
           User对象
```

### 5.3 BaseTypeHandler 抽象基类

```java
public abstract class BaseTypeHandler<T> implements TypeHandler<T> {
    // 空值处理模板方法
    public void setParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) {
        if (parameter == null) {
            ps.setNull(i, jdbcType.TYPE_CODE);  // JDBC处理NULL
        } else {
            setNonNullParameter(ps, i, parameter, jdbcType);  // 子类实现
        }
    }
    
    protected abstract void setNonNullParameter(...);
    protected abstract T getNullableResult(...);
}
```

**模板方法模式**：
- 父类处理空值判断
- 子类处理具体转换逻辑

### 5.4 具体处理器示例

**StringTypeHandler**:
```java
public class StringTypeHandler extends BaseTypeHandler<String> {
    protected void setNonNullParameter(PreparedStatement ps, int i, 
            String parameter, JdbcType jdbcType) {
        ps.setString(i, parameter);  // JDBC setString
    }
    
    protected String getNullableResult(ResultSet rs, String columnName) {
        return rs.getString(columnName);  // JDBC getString
    }
}
```

**IntegerTypeHandler**:
```java
public class IntegerTypeHandler extends BaseTypeHandler<Integer> {
    protected void setNonNullParameter(PreparedStatement ps, int i, 
            Integer parameter, JdbcType jdbcType) {
        ps.setInt(i, parameter);
    }
    
    protected Integer getNullableResult(ResultSet rs, String columnName) {
        return rs.getInt(columnName);
    }
}
```

### 5.5 TypeHandlerRegistry - 处理器注册表

**核心数据结构**：
```java
public final class TypeHandlerRegistry {
    // JDBC类型 → 处理器
    private final Map<JdbcType, TypeHandler<?>> JDBC_TYPE_HANDLER_MAP;
    
    // Java类型 → (JDBC类型 → 处理器)
    private final Map<Type, Map<JdbcType, TypeHandler<?>>> TYPE_HANDLER_MAP;
    
    // 处理器Class → 处理器实例
    private final Map<Class<?>, TypeHandler<?>> ALL_TYPE_HANDLERS_MAP;
}
```

**注册逻辑**：
```java
// 构造函数中批量注册
register(Integer.class, new IntegerTypeHandler());
register(String.class, new StringTypeHandler());
register(LocalDate.class, new LocalDateTypeHandler());
// ... 共27个内置处理器
```

**查询流程**（getTypeHandler）：

```
getTypeHandler(Integer.class, JdbcType.INTEGER)
       ↓
1. TYPE_HANDLER_MAP.get(Integer.class)
       ↓
2. jdbcHandlerMap.get(INTEGER)
       ↓
3. 如无，检查父类/接口
       ↓
4. JDBC_TYPE_HANDLER_MAP.get(INTEGER)
       ↓
5. 返回 DEFAULT_TYPE_HANDLER (兜底)
```

### 5.8 学习要点

**类型转换核心原理**：

```java
// Java → JDBC 类型映射
User.id (Integer)
    ↓ IntegerTypeHandler
ps.setInt(index, value)

// JDBC → Java 类型映射
rs.getInt(column)
    ↓ IntegerTypeHandler
User.setId(value)
```

**类型匹配规则**：
```
优先级：
1. 精确匹配 (Integer, INTEGER)
2. 无JdbcType匹配 (Integer)
3. 父类匹配
4. 接口匹配
5. 枚举特判
6. JDBC类型兜底
7. ObjectTypeHandler最终兜底
```

## 8. 面试常问题目

### Q1: TypeHandler 如何实现类型转换？
**Answer**: 通过继承BaseTypeHandler，实现setNonNullParameter和getNullableResult两个抽象方法，分别处理PreparedStatement的参数设置和ResultSet的结果读取。

### Q2: #{} 和 ${} 的区别？
**Answer**: #{} 使用预编译，生成?占位符，防止SQL注入；${} 直接字符串拼接，存在SQL注入风险。

### Q3: TypeHandlerRegistry 的查找顺序？
**Answer**: 精确匹配 → 无JdbcType匹配 → 父类/接口 → JDBC类型兜底 → ObjectTypeHandler最终兜底。

### Q4: 为什么需要TypeHandler？
**Answer**: JDBC的setInt/getString等方法不能自动适配所有Java类型，需要类型处理器在Java类型和JDBC类型之间做双向转换。
