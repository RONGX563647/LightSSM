# Light Framework 开发文档

## 项目概述

**Light Framework** 是一个轻量级SSM框架，完整实现了IoC容器、SpringMVC、ORM和AOP核心功能，适合用于学习Spring框架原理和简历展示。

## 技术架构

### 1. IoC/DI容器模块

#### 核心设计

```
ApplicationContext (应用上下文)
    ↓
DefaultListableBeanFactory (核心Bean工厂)
    ├── singletonObjects (一级缓存：完整Bean实例)
    ├── earlySingletonObjects (二级缓存：早期Bean实例)
    └── singletonFactories (三级缓存：Bean工厂对象)
```

#### 三级缓存解决循环依赖

**实现原理**：
1. **一级缓存** (`singletonObjects`)：存储完全初始化好的Bean实例
2. **二级缓存** (`earlySingletonObjects`)：存储早期暴露的Bean实例（解决循环依赖）
3. **三级缓存** (`singletonFactories`)：存储Bean工厂对象，用于创建早期Bean

**流程**：
```
A依赖B，B依赖A
1. 创建A实例，暴露到三级缓存
2. A注入B，发现B未创建
3. 创建B实例，暴露到三级缓存
4. B注入A，从三级缓存获取A的早期引用
5. B完成初始化，放入一级缓存
6. A完成初始化，放入一级缓存
```

#### 核心类

| 类名 | 功能 |
|------|------|
| `ApplicationContext` | 应用上下文接口 |
| `AnnotationConfigApplicationContext` | 注解配置应用上下文 |
| `DefaultListableBeanFactory` | 核心Bean工厂实现 |
| `BeanDefinition` | Bean定义信息 |
| `ClassPathBeanDefinitionScanner` | 包扫描器 |

#### 注解支持

- `@Component`：组件注解，标记为Spring管理的Bean
- `@Autowired`：自动注入注解，支持required属性
- `@Scope`：作用域注解，支持singleton/prototype

### 2. SpringMVC模块

#### 核心流程

```
HTTP请求
    ↓
DispatcherServlet (核心分发器)
    ↓
HandlerMapping (处理器映射)
    ↓ 找到对应的Handler
HandlerExecutionChain (执行链)
    ↓ 包含拦截器
HandlerAdapter (处理器适配)
    ↓ 执行Handler方法
ModelAndView (模型视图)
    ↓
ViewResolver (视图解析)
    ↓
View.render() (视图渲染)
    ↓
HTTP响应
```

#### 核心类

| 类名 | 功能 |
|------|------|
| `DispatcherServlet` | 核心请求分发器 |
| `RequestMappingHandlerMapping` | @RequestMapping处理器映射 |
| `RequestMappingHandlerAdapter` | 处理器适配器 |
| `HandlerExecutionChain` | 执行链（包含拦截器） |
| `InternalResourceViewResolver` | JSP视图解析器 |
| `JsonView` | JSON视图渲染 |

#### 注解支持

- `@RequestMapping`：请求映射注解
- `@ResponseBody`：响应体注解，返回JSON
- `@RequestParam`：请求参数绑定
- `@PathVariable`：路径变量绑定

### 3. ORM模块

#### 核心架构

```
SqlSessionFactoryBuilder
    ↓ 解析XML配置
Configuration (全局配置)
    ↓
SqlSessionFactory
    ↓ openSession()
SqlSession (会话入口)
    ↓ getMapper()
MapperProxy (JDK动态代理)
    ↓ invoke()
Executor (执行器)
    ↓
StatementHandler (语句处理器)
    ↓ JDBC执行
ResultSetHandler (结果映射)
```

#### 核心功能

1. **XML SQL映射**：支持XML配置SQL语句
2. **动态SQL**：支持`<if>`、`<trim>`等标签，使用OGNL表达式
3. **参数绑定**：自动将参数绑定到SQL语句
4. **结果映射**：自动将ResultSet映射为POJO
5. **插件机制**：基于拦截器链的插件扩展

#### 核心类

| 类名 | 功能 |
|------|------|
| `SqlSessionFactory` | 会话工厂 |
| `SqlSession` | 会话接口 |
| `MapperProxy` | Mapper代理 |
| `SimpleExecutor` | SQL执行器 |
| `DefaultResultSetHandler` | 结果集处理器 |

### 4. AOP模块

#### 核心设计

```
目标对象
    ↓
ProxyFactory (代理工厂)
    ↓ 判断是否有接口
JdkDynamicAopProxy / CglibAopProxy
    ↓ 创建代理对象
MethodInvocation (方法调用)
    ↓ 责任链模式
MethodInterceptor链 (拦截器链)
    ├── BeforeMethodInterceptor
    ├── AroundMethodInterceptor
    └── AfterMethodInterceptor
    ↓
目标方法执行
```

#### JDK/CGLIB双代理机制

**选择策略**：
- 如果目标类实现了接口 → 使用JDK动态代理
- 如果目标类未实现接口 → 使用CGLIB代理

#### 核心类

| 类名 | 功能 |
|------|------|
| `JdkDynamicAopProxy` | JDK动态代理实现 |
| `CglibAopProxy` | CGLIB代理实现 |
| `ProxyFactory` | 代理工厂 |
| `MethodInvocation` | 方法调用对象 |
| `ProceedingJoinPoint` | 可执行的连接点 |

#### 注解支持

- `@Aspect`：切面注解
- `@Before`：前置通知
- `@After`：后置通知
- `@Around`：环绕通知
- `@Pointcut`：切入点表达式

## 设计模式应用

| 设计模式 | 应用位置 | 说明 |
|----------|----------|------|
| **工厂模式** | SqlSessionFactory、BeanFactory | 创建复杂对象 |
| **代理模式** | MapperProxy、JdkDynamicAopProxy | 控制对象访问 |
| **模板方法** | BaseExecutor | 定义执行骨架 |
| **建造者模式** | MappedStatement.Builder | 分步构建对象 |
| **策略模式** | TypeHandler | 类型转换策略 |
| **责任链模式** | InterceptorChain、MethodInterceptor | 链式处理 |
| **单例模式** | ApplicationContext | 全局唯一实例 |
