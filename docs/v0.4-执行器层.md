# v0.4 执行器层

## 版本信息
- **版本号**: v0.4
- **提交哈希**: 148e74c

## 模块概述

执行器层是ORM框架的核心，负责SQL的真正执行。本模块包含：

1. **Executor** - 执行器接口
2. **BaseExecutor** - 执行器基类
3. **SimpleExecutor** - 简单执行器
4. **StatementHandler** - 语句处理器
5. **PreparedStatementHandler** - 预处理语句处理器
6. **ResultSetHandler** - 结果集处理器
7. **ParameterHandler** - 参数处理器
8. **KeyGenerator** - 主键生成器

---

## 1. 整体架构

### 1.1 执行流程图

```
用户代码
   ↓
SqlSession.selectOne()
   ↓
Executor.query()
   ├─→ StatementHandler.prepare()        // 创建语句
   ├─→ ParameterHandler.setParameters() // 设置参数
   ├─→ PreparedStatement.execute() // 执行SQL
   ├─→ ResultSetHandler.handleResultSets() // 处理结果
   ↓
返回结果对象
```

### 1.2 责任链模式

Executor → StatementHandler → ParameterHandler → ResultSetHandler

每一层专注自己的职责，符合单一职责原则。

## 2. Executor - 执行器接口

### 2.1 接口定义

```java
public interface Executor {
    // 更新操作
    int update(MappedStatement ms, Object parameter);
    
    // 查询操作
    <E> List<E> query(MappedStatement ms, Object parameter, 
                      RowBounds rowBounds, ResultHandler resultHandler);
    
    // 事务管理
    Transaction getTransaction();
    void commit(boolean required);
    void rollback(boolean required);
    
    // 关闭
    void close(boolean forceRollback);
}
```

### 2.2 方法分类

| 方法 | 说明 |
|------|------|
| update | INSERT/UPDATE/DELETE |
| query | SELECT |
| commit | 提交事务 |
| rollback | 回滚事务 |
| close | 关闭执行器 |

## 3. BaseExecutor - 执行器基类

### 3.1 模板方法模式

```java
public abstract class BaseExecutor implements Executor {
    protected Configuration configuration;
    protected Transaction transaction;
    protected boolean closed;
    
    // 模板方法 - query
    @Override
    public <E> List<E> query(MappedStatement ms, Object parameter, 
                          RowBounds rowBounds, ResultHandler resultHandler) {
        // 1. 获取BoundSql（处理动态SQL）
        BoundSql boundSql = ms.getBoundSql(parameter);
        // 2. 模板方法
        return doQuery(ms, parameter, rowBounds, resultHandler, boundSql);
    }
    
    // 子类实现
    protected abstract <E> List<E> doQuery(...);
}
```

### 3.2 核心职责

1. ** BoundSql 生成** - 调用SqlSource获取SQL
2. ** 模板方法** - 定义查询流程
3. ** 事务管理** - commit/rollback/close

### 3.3 源码分析

```java
@Override
public <E> List<E> query(MappedStatement ms, Object parameter, 
                        RowBounds rowBounds, ResultHandler resultHandler) {
    // 1. 检查执行器是否关闭
    if (closed) {
        throw new RuntimeException("Executor was closed.");
    }
    // 2. 获取BoundSql
    BoundSql boundSql = ms.getBoundSql(parameter);
    // 3. 调用子类实现
    return doQuery(ms, parameter, rowBounds, resultHandler, boundSql);
}

@Override
public void commit(boolean required) {
    if (!closed) {
        if (required) {
            transaction.commit();  // 提交事务
        }
    }
}

@Override
public void close(boolean forceRollback) {
    try {
        try {
            rollback(forceRollback);  // 回滚
        } finally {
            transaction.close();    // 关闭连接
        }
    } finally {
        closed = true;
    }
}
```

### 3.4 事务控制

```java
// commit流程
public void commit(boolean required) {
    if (closed) throw new RuntimeException("Cannot commit");
    if (required) {
        transaction.commit();  // 实际提交
    }
}

// rollback流程  
public void rollback(boolean required) {
    if (!closed) {
        if (required) {
            transaction.rollback();  // 实际回滚
        }
    }
}
```

**required参数**：
- `true` - 强制提交/回滚
- `false` - 仅关闭

## 4. SimpleExecutor - 简单执行器

### 4.1 实现

```java
public class SimpleExecutor extends BaseExecutor {
    protected <E> List<E> doQuery(MappedStatement ms, Object parameter,
                                 RowBounds rowBounds, ResultHandler resultHandler,
                                 BoundSql boundSql) throws SQLException {
        // 1. 获取数据库连接
        Connection connection = transaction.getConnection();
        
        // 2. 创建StatementHandler
        StatementHandler handler = configuration.newStatementHandler(
            this, ms, parameter, rowBounds, resultHandler, boundSql);
        
        // 3. 准备语句
        Statement statement = handler.prepare(connection);
        
        // 4. 设置参数
        handler.parameterize(statement);
        
        // 5. 执行查询
        return handler.query(statement, resultHandler);
    }
    
    protected int doUpdate(MappedStatement ms, Object parameter) {
        // 类似流程，执行更新
    }
}
```

### 4.2 执行流程图

```
doQuery()
       ↓
1. transaction.getConnection()  获取连接
       ↓
2. newStatementHandler()        创建处理器
       ↓
3. handler.prepare()          准备语句
       ↓
4. handler.parameterize()     设置参数
       ↓
5. handler.query()           执行查询
       ↓
返回结果
```

## 5. StatementHandler - 语句处理器

### 5.1 接口定义

```java
public interface StatementHandler {
    // 准备语句（创建Statement）
    Statement prepare(Connection connection);
    
    // 参数化（设置参数）
    void parameterize(Statement statement);
    
    // 执行更新
    int update(Statement statement);
    
    // 执行查询
    <E> List<E> query(Statement statement, ResultHandler resultHandler);
    
    // 获取BoundSql
    BoundSql getBoundSql();
}
```

### 5.2 方法职责

| 方法 | 说明 |
|------|------|
| prepare | 创建PreparedStatement |
| parameterize | 设置参数到Statement |
| update | 执行UPDATE/INSERT/DELETE |
| query | 执行SELECT |

### 5.3 PreparedStatementHandler

```java
public class PreparedStatementHandler extends BaseStatementHandler {
    @Override
    protected Statement instantiateStatement(Connection connection) {
        // 创建预编译语句
        return connection.prepareStatement(boundSql.getSql());
    }
    
    @Override
    public void parameterize(Statement statement) {
        // 委托给ParameterHandler
        parameterHandler.setParameters((PreparedStatement) statement);
    }
    
    @Override
    public int update(Statement statement) {
        PreparedStatement ps = (PreparedStatement) statement;
        ps.execute();  // 执行
        int rows = ps.getUpdateCount();
        // 处理主键
        keyGenerator.processAfter(executor, ms, ps, parameterObject);
        return rows;
    }
    
    @Override
    public <E> List<E> query(Statement statement, ResultHandler rh) {
        PreparedStatement ps = (PreparedStatement) statement;
        ps.execute();  // 执行
        return resultSetHandler.handleResultSets(ps);  // 处理结果
    }
}
```

## 6. ParameterHandler - 参数处理器

### 6.1 作用

负责将Java对象设置到PreparedStatement中：

```
User{id=1, name="Tom"} 
       ↓ ParameterHandler
PreparedStatement: SET id=1, name='Tom'
```

### 6.2 接口

```java
public interface ParameterHandler {
    void setParameters(PreparedStatement ps);
}
```

### 6.3 DefaultParameterHandler

```java
public class DefaultParameterHandler implements ParameterHandler {
    private final TypeHandlerRegistry typeHandlerRegistry;
    
    @Override
    public void setParameters(PreparedStatement ps) {
        // 遍历所有参数映射
        for (ParameterMapping mapping : parameterMappings) {
            // 1. 获取值
            Object value = metaObject.getValue(mapping.getProperty());
            
            // 2. 获取类型处理器
            TypeHandler<?> typeHandler = mapping.getTypeHandler();
            
            // 3. 设置参数
            if (value == null) {
                ps.setNull(i, mapping.getJdbcType().TYPE_CODE);
            } else {
                typeHandler.setParameter(ps, i, value, mapping.getJdbcType());
            }
        }
    }
}
```

### 6.4 参数设置流程

```
parameterObject (User对象)
       ↓
MetaObject包装
       ↓
遍历parameterMappings
       ↓
获取每个属性值
       ↓
获取对应TypeHandler
       ↓
TypeHandler.setParameter()
       ↓
PreparedStatement.setXxx()
```

## 7. ResultSetHandler - 结果集处理器

### 7.1 核心职责

将JDBC ResultSet转换为Java对象：

```
ResultSet (JDBC)
       ↓ DefaultResultSetHandler
User对象 (Java)
```

### 7.2 DefaultResultSetHandler

```java
public class DefaultResultSetHandler implements ResultSetHandler {
    public List<Object> handleResultSets(Statement stmt) {
        List<Object> results = new ArrayList<>();
        
        // 遍历ResultMap
        while (resultMaps.size() > resultSetCount) {
            ResultMap rm = resultMaps.get(resultSetCount);
            // 处理每行数据
            handleRowValuesForSimpleResultMap(rsw, rm, handler);
            results.add(handler.getResultList());
        }
        return results;
    }
}
```

### 7.3 行处理流程

```java
private void handleRowValuesForSimpleResultMap(...) {
    ResultSet rs = rsw.getResultSet();
    while (resultSet.next()) {
        // 1. 创建对象
        Object rowValue = createResultObject(...);
        
        // 2. 自动映射 (列名→属性名)
        applyAutomaticMappings(rsw, resultMap, metaObject, ...);
        
        // 3. 显式映射 (resultMap配置)
        applyPropertyMappings(rsw, resultMap, metaObject, ...);
        
        // 4. 添加到结果集
        resultHandler.handleResult(resultContext);
    }
}
```

### 7.4 结果映射详解

#### 7.4.1 创建结果对象

```java
private Object createResultObject(...) {
    Class<?> resultType = resultMap.getType();
    
    // 基本类型 (Integer, String等)
    if (typeHandlerRegistry.hasTypeHandler(resultType)) {
        return createPrimitiveResultObject(...);
    }
    // Bean对象
    else if (metaType.hasDefaultConstructor()) {
        return objectFactory.create(resultType);
    }
}
```

#### 7.4.2 自动映射

```java
private boolean applyAutomaticMappings(...) {
    // 找出ResultSet中未映射的列
    List<String> unmappedColumns = rsw.getUnmappedColumnNames(...);
    
    for (String column : unmappedColumns) {
        // 查找对应属性
        String property = metaObject.findProperty(column);
        if (property != null && metaObject.hasSetter(property)) {
            // 获取TypeHandler
            TypeHandler<?> handler = rsw.getTypeHandler(propertyType, column);
            // 获取值
            Object value = handler.getResult(rs, column);
            // 设置值
            metaObject.setValue(property, value);
        }
    }
}
```

**自动映射原理**：
- ResultSet的列名与JavaBean属性名自动匹配
- 列名"USER_NAME" → 属性"userName"

#### 7.4.3 显式映射

```java
private boolean applyPropertyMappings(...) {
    // 使用resultMap中配置的映射
    for (ResultMapping mapping : resultMap.getPropertyMappings()) {
        String column = mapping.getColumn();
        String property = mapping.getProperty();
        
        // 获取值
        Object value = typeHandler.getResult(rs, column);
        
        // 设置值
        metaObject.setValue(property, value);
    }
}
```

**显式映射**：
- 通过resultMap配置指定映射关系
- column → property

### 7.5 结果映射流程图

```
ResultSet
   ↓
handleResultSets()
   ↓
遍历ResultMap
   ↓
handleRowValuesForSimpleResultMap()
   ↓
while(resultSet.next())
   ├─→ createResultObject()      创建对象
   ├─→ applyAutomaticMappings() 自动映射
   ├─→ applyPropertyMappings()  显式映射
   └─→ handleResult()         保存结果
```

## 8. KeyGenerator - 主键生成器

### 8.1 接口

```java
public interface KeyGenerator {
    void processBefore(Executor executor, MappedStatement ms, 
                     PreparedStatement ps, Object parameter);
    
    void processAfter(Executor executor, MappedStatement ms,
                  PreparedStatement ps, Object parameter);
}
```

### 8.2 类型

| 类型 | 说明 | 适用数据库 |
|------|------|----------|
| Jdbc3KeyGenerator | JDBC3.0自增 | MySQL |
| SelectKeyGenerator | SELECT语句 | Oracle |
| NoKeyGenerator | 无主键 | - |

### 8.3 Jdbc3KeyGenerator

```java
public class Jdbc3KeyGenerator implements KeyGenerator {
    @Override
    public void processAfter(Executor executor, MappedStatement ms,
                          PreparedStatement ps, Object parameter) {
        // 获取自增主键
        ResultSet rs = ps.getGeneratedKeys();
        // 设置回parameter对象
        metaObject.setValue(id, rs.getLong(1));
    }
}
```

### 8.4 使用场景

**MySQL自增**：
```xml
<insert id="insertUser" useGeneratedKeys="true" keyProperty="id">
    INSERT INTO user (name) VALUES (#{name})
</insert>
```

**Oracle序列**：
```xml
<insert id="insertUser">
    <selectKey keyProperty="id" resultType="long">
        SELECT seq.nextval FROM dual
    </selectKey>
    INSERT INTO user (id, name) VALUES (#{id}, #{name})
</insert>
```

### 9.3 内存分页 vs SQL分页

| 方式 | 说明 | 性能 |
|------|------|------|
| 内存分页 | 先查全部，再内存截取 | 差 |

## 13. 面试问题

### Q1: Executor的职责？
**Answer**: 负责SQL的实际执行，协调StatementHandler、ParameterHandler、ResultSetHandler完成数据库操作。

### Q2: ParameterHandler的作用？
**Answer**: 将Java对象的属性值设置到PreparedStatement中，处理类型转换。

### Q3: ResultSetHandler如何将ResultSet转为对象？
**Answer**: 1.创建对象 2.自动映射 3.显式映射。

### Q4: 动态SQL在哪里处理？
**Answer**: 在Executor.query()中调用ms.getBoundSql(parameter)时处理。
