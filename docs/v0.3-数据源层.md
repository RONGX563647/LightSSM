## 模块概述

数据源层负责数据库连接的获取和管理,是ORM框架与数据库的桥梁。本模块包含：

1. **DataSourceFactory** - 数据源工厂
2. **UnpooledDataSource** - 非池化数据源
3. **PooledDataSource** - 连接池实现
4. **DruidDataSourceFactory** - Druid集成

## 1. JDBC DataSource 规范

### 1.1 javax.sql.DataSource 接口

JDBC标准定义的数据源接口：

```java
public interface DataSource extends CommonDataSource {
    Connection getConnection() throws SQLException;
    Connection getConnection(String username, String password) throws SQLException;
}
```

### 1.2 为什么需要DataSource？

**传统方式（DriverManager）**：
```java
// 每次请求都创建新连接
Connection conn = DriverManager.getConnection(url, user, pwd);
```
问题：创建连接开销大，频繁创建/销毁效率低

**连接池方式**：
```java
// 从连接池获取复用连接
Connection conn = dataSource.getConnection();
```
优势：连接复用，减少开销

## 2. UnpooledDataSource - 非池化数据源

### 2.1 实现原理

```java
public class UnpooledDataSource implements DataSource {
    private String driver;
    private String url;
    private String username;
    private String password;
    
    @Override
    public Connection getConnection() throws SQLException {
        return getConnection(username, password);
    }
    
    @Override
    public Connection getConnection(String user, String pwd) throws SQLException {
        // 每次都创建新连接
        Connection = DriverManager.getConnection(url, user, pwd);
        return Connection;
    }
}
```

### 2.2 特点

| 特性 | 说明 |
|------|------|
| 每次调用 | 创建新连接 |
| 线程安全 | 无共享状态 |
| 适用场景 | 测试、低并发 |

### 2.3 源码分析

```java
// 获取连接的核心逻辑
protected Connection getConnection(String username, String password) throws SQLException {
    // 1. 加载驱动（如果未加载）
    registerDriver();
    
    // 2. 通过DriverManager获取连接
    return DriverManager.getConnection(url, username, password);
}

private void registerDriver() {
    if (!registered) {
        // 动态加载JDBC驱动
        Class.forName(driver);
        registered = true;
    }
}
```

## 3. PooledDataSource - 连接池实现（重点）

### 3.1 核心概念

连接池通过复用连接来提高性能：

```
不使用连接池:
请求1 → 创建连接 → 使用 → 关闭连接
请求2 → 创建连接 → 使用 → 关闭连接
请求3 → 创建连接 → 使用 → 关闭连接
          ↑ 每次都创建

使用连接池:
请求1 → 获取连接 → 使用 → 归还连接池
请求2 → 获取连接 → 使用 → 归还连接池
请求3 → 获取连接 → 使用 → 归还连接池
          ↑ 复用已有连接
```

### 3.2 核心数据结构

```java
public class PooledDataSource implements DataSource {
    // 池状态
    private final PoolState state = new PoolState(this);
    
    // 底层数据源
    private final UnpooledDataSource dataSource;
    
    // 配置参数
    protected int poolMaximumActiveConnections = 10;    // 最大活跃连接数
    protected int poolMaximumIdleConnections = 5;       // 最大空闲连接数
    protected int poolMaximumCheckoutTime = 20000;    // 最大检出时间(ms)
    protected int poolTimeToWait = 20000;             // 等待超时(ms)
    protected String poolPingQuery = "NO PING QUERY SET"; // ping查询
    protected boolean poolPingEnabled = false;         // 是否开启ping
}
```

### 3.3 PoolState - 池状态

```java
public class PoolState {
    // 活跃连接列表
    public final List<PooledConnection> activeConnections = new ArrayList<>();
    
    // 空闲连接列表
    public final List<PooledConnection> idleConnections = new ArrayList<>();
    
    // 统计指标
    public long hadToWaitCount = 0;           // 等待次数
    public long badConnectionCount = 0;         // 坏连接数
    public long claimedOverdueConnectionCount = 0; // 强制回收次数
    public long accumulatedWaitTime = 0;       // 总等待时间
    public long accumulatedCheckoutTime = 0;   // 总检出时间
}
```

### 3.4 连接获取流程 - popConnection

```java
private PooledConnection popConnection(String username, String password) {
    // 1. 有空闲连接 → 直接复用
    if (!state.idleConnections.isEmpty()) {
        conn = state.idleConnections.remove(0);
    }
    // 2. 无空闲，但活跃数未满 → 创建新连接
    else if (state.activeConnections.size() < poolMaximumActiveConnections) {
        conn = new PooledConnection(dataSource.getConnection(), this);
    }
    // 3. 活跃数已满 → 尝试复用最老的连接
    else {
        // 获取最老的活跃连接
        PooledConnection oldest = state.activeConnections.get(0);
        long checkoutTime = oldest.getCheckoutTime();
        
        // 如果检出时间过长，强制回收
        if (checkoutTime > poolMaximumCheckoutTime) {
            conn = new PooledConnection(oldest.getRealConnection(), this);
            oldest.invalidate();
        }
        // 否则等待
        else {
            state.wait(poolTimeToWait);  // 等待其他线程释放
        }
    }
}
```

### 3.5 连接归还流程 - pushConnection

```java
protected void pushConnection(PooledConnection connection) {
    synchronized (state) {
        // 1. 从活跃列表移除
        state.activeConnections.remove(connection);
        
        // 2. 检查连接有效性
        if (connection.isValid()) {
            // 3. 空闲数未满 → 加入空闲队列
            if (state.idleConnections.size() < poolMaximumIdleConnections) {
                state.idleConnections.add(newConnection);
            }
            // 4. 空闲数已满 → 关闭连接
            else {
                connection.getRealConnection().close();
            }
        }
        // 5. 无效连接 → 丢弃
        else {
            state.badConnectionCount++;
        }
    }
}
```

### 3.6 连接有效性检测 - pingConnection

```java
protected boolean pingConnection(PooledConnection conn) {
    // 1. 检查连接是否关闭
    if (!conn.getRealConnection().isClosed()) {
        return false;
    }
    
    // 2. 如果开启了ping
    if (poolPingEnabled) {
        // 检查空闲时间
        if (conn.getTimeElapsedSinceLastUse() > poolPingConnectionsNotUsedFor) {
            // 执行ping查询
            Statement stmt = conn.createStatement();
            ResultSet rs = stmt.executeQuery(poolPingQuery);
            rs.close();
        }
    }
    return true;
}
```

### 3.7 连接代理 - PooledConnection

```java
public class PooledConnection implements Connection {
    private Connection realConnection;  // 真实连接
    private Connection proxyConnection; // 代理连接
    private PooledDataSource dataSource;
    private long createdTimestamp;
    private long lastUsedTimestamp;
    private long checkoutTime;
    private boolean valid;
    
    public Connection getProxyConnection() {
        // 返回代理对象，拦截close()调用
        return proxyConnection;
    }
}
```

**代理机制**：
```
用户调用 connection.close()
       ↓
Proxy.invoke()
       ↓
PooledConnection.close()
       ↓
dataSource.pushConnection(this)  // 归还连接池
       ↓
而不是真正关闭连接
```

## 5. Druid集成

### 5.1 为什么集成Druid？

Druid是阿里巴巴开源的数据库连接池，相比简单实现：

| 特性 | PooledDataSource | Druid |
|------|-----------------|------|
| 连接检测 | 简单 | 完善 |
| 监控 | 无 | 丰富 |
| 防盗链 | 无 | 支持 |
| 诊断 | 无 | 支持 |

### 5.2 DruidDataSourceFactory

```java
public class DruidDataSourceFactory implements DataSourceFactory {
    private DruidDataSource dataSource;
    
    @Override
    public void setProperties(Properties props) {
        dataSource = new DruidDataSource();
        dataSource.setUrl(props.getProperty("url"));
        dataSource.setUsername(props.getProperty("username"));
        dataSource.setPassword(props.getProperty("password"));
    }
    
    @Override
    public DataSource getDataSource() {
        return dataSource;
    }
}
```

## 8. 核心流程图

### 8.1 获取连接流程

```
getConnection()
       ↓
synchronized(state)
       ↓
有空闲连接?
  ├─ 是 → remove(0) → 检查有效性 → 设置checkoutTime → 加入active → 返回
  │
  ├─ 否 → 活跃数<最大值?
  │         ├─ 是 → new Connection() → 创建 → 加入active → 返回
  │         │
  │         ├─ 否 → 最老连接已超时?
  │                    ├─ 是 → 强制回收 → 创建新连接 → 返回
  │                    │
  │                    ├─ 否 → wait(poolTimeToWait) → 重试
```

### 8.2 归还连接流程

```
connection.close()
       ↓
Proxy.invoke(close)
       ↓
PooledConnection.close()
       ↓
pushConnection(this)
       ↓
synchronized(state)
       ↓
valid?
  ├─ 是 → idleConnections < poolMaximumIdleConnections?
  │         ├─ 是 → 加入idleConnections
  │         ├─ 否 → 真正关闭连接
  │
  ├─ 否 → 坏连接计数+1
       ↓
notifyAll()  // 唤醒等待线程
```

## 9. 性能优化建议

### 9.1 连接池大小设置

```
公式: 最佳连接数 = CPU核心数 * 2 + 磁盘数

例如: 4核CPU + 1磁盘 = 9个连接
```

### 9.2 监控指标

```java
// 查看池状态
hadToWaitCount      // 等待次数（高=池太小）
badConnectionCount // 坏连接数（高=数据库问题）
accumulatedWaitTime // 总等待时间
```

## 10. 面试问题

### Q1: 连接池的作用？
**Answer**: 复用数据库连接，减少连接创建开销，提高性能。

### Q2: 连接池如何保证线程安全？
**Answer**: 使用synchronized同步对连接列表的操作，或使用更高效的并发集合。

### Q3: 为什么需要代理Connection？
**Answer**: 拦截close()方法，实现"假关闭"——归还连接而非真正关闭。

### Q4: 连接池满时如何处理？
**Answer**: 等待超时后可强制回收最老的连接，或抛出异常。

### Q5: 如何检测连接有效性？
**Answer**: 心跳检测(ping)，定期执行SQL验证连接是否可用。

## 11. 总结

数据源层要点：
1. **连接池**复用连接是性能关键
2. **代理模式**实现连接的"假关闭"
3. **synchronized**保证线程安全
4. **配置参数**影响性能和稳定性

连接池是数据库访问性能的基础，选择合适的连接池和参数配置非常重要。
