# 06 - 数据源与事务管理实现

## 1. 数据源架构

```java
}
```

**MyBatis官方DataSourceFactory**：

### 1.2 数据源类型

## 2. 非池化数据源

### 2.1 UnpooledDataSource

**核心实现**：
```java
public class UnpooledDataSource implements DataSource {
    private String driver;
    private String url;
    private String username;
    private String password;
    
    @Override
    public Connection getConnection() throws SQLException {
        return doGetConnection(username, password);
    }
    
    private Connection doGetConnection(String username, String password) throws SQLException {
        Properties props = new Properties();
        if (driver != null) props.setProperty("driver", driver);
        props.putAll(driverProperties);
        
        if (username != null) props.setProperty("user", username);
        if (password != null) props.setProperty("password", password);
        
        return DriverManager.getConnection(url, props);
    }
}
```

### 2.2 与MyBatis官方对比

**MyBatis官方的UnpooledDataSource**：
- 支持JNDI数据源查找
- 支持设置默认事务隔离级别
- 支持日志记录
- 支持驱动注册

**LightSSM简化**：
- 仅支持基本的DriverManager获取连接
- 不支持JNDI
- 不支持事务隔离级别配置

## 3. 池化数据源

### 3.1 PooledDataSource实现

**LightSSM实现**：
```java
public class PooledDataSource implements DataSource {
    private DataSourceFactory dataSourceFactory;
    private int poolMaximumActiveConnections = 10;
    private int poolMaximumIdleConnections = 5;
    private int poolMaximumCheckoutTime = 20000;
    private int poolTimeToWait = 20000;
    
    private final List<PooledConnection> idleConnections = new ArrayList<>();
    private final List<PooledConnection> activeConnections = new ArrayList<>();
    
    @Override
    public Connection getConnection() throws SQLException {
        return getConnectionFromPool();
    }
    
    private Connection getConnectionFromPool() throws SQLException {
        synchronized (this) {
            // 1. 查找空闲连接
            if (!idleConnections.isEmpty()) {
                PooledConnection pooledConn = idleConnections.remove(0);
                activeConnections.add(pooledConn);
                return pooledConn.getRealConnection();
            }
            
            // 2. 创建新连接（未达上限）
            if (activeConnections.size() < poolMaximumActiveConnections) {
                Connection realConn = dataSourceFactory.getDataSource().getConnection();
                PooledConnection pooledConn = new PooledConnection(realConn, this);
                activeConnections.add(pooledConn);
                return pooledConn.getRealConnection();
            }
            
            // 3. 等待可用连接
            try {
                this.wait(poolTimeToWait);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            
            return getConnectionFromPool(); // 重试
        }
    }
}
```

### 3.2 PooledConnection代理连接

```java
public class PooledConnection implements InvocationHandler {
    private Connection realConnection;
    private PooledDataSource dataSource;
    private boolean closed = false;
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        if ("close".equals(method.getName())) {
            // 调用close时，不真正关闭，而是返回连接池
            returnReturnToPool();
            return null;
        }
        return method.invoke(realConnection, args);
    }
    
    private void returnToPool() throws SQLException {
        synchronized (dataSource) {
            dataSource.activeConnections.remove(this);
            dataSource.idleConnections.add(this);
            closed = false;
            dataSource.notifyAll(); // 唤醒等待的线程
        }
    }
}
```

### 3.3 与MyBatis官方对比

**MyBatis官方的PooledDataSource**：
- 使用动态代理（Proxy.newProxyInstance）
- 支持连接池统计（PooledState）
- 支持连接超时检测
- 支持Ping测试连接有效性
- 支持最大连接数限制

**LightSSM简化**：
- 基础的连接获取和归还
- 简单的空闲/活跃列表管理
- 基础的等待机制

## 4. Druid数据源集成

### 4.1 DruidDataSourceFactory

**LightSSM实现**：
```java
public class DruidDataSourceFactory implements DataSourceFactory {
    private Properties properties;
    
    @Override
    public void setProperties(Properties props) {
        this.properties = props;
    }
    
    @Override
    public DataSource getDataSource() {
        try {
            DruidDataSource dataSource = new DruidDataSource();
            dataSource.setDriverClassName(properties.getProperty("driver"));
            dataSource.setUrl(properties.getProperty("url"));
            dataSource.setUsername(properties.getProperty("username"));
            dataSource.setPassword(properties.getProperty("password"));
            
            // 配置连接池参数
            dataSource.setInitialSize(
                Integer.parseInt(properties.getProperty("initialSize", "5")));
            dataSource.setMinIdle(
                Integer.parseInt(properties.getProperty("minIdle", "10")));
            dataSource.setMaxActive(
                Integer.parseInt(properties.getProperty("maxActive", "20")));
            
            return dataSource;
        } catch (Exception e) {
            throw new RuntimeException("Failed to create DruidDataSource", e);
        }
    }
}
```

### 4.2 与MyBatis官方对比

**MyBatis官方**不直接集成Druid，而是通过SPI机制：
```java
// 用户可以自定义DataSourceFactory实现
public class MyDruidDataSourceFactory implements DataSourceFactory {
    // 自定义实现
}
```

**LightSSM**直接内置Druid支持，方便使用。

## 5. 事务管理

### 5.1 Transaction接口

**LightSSM实现**：
```java
public interface Transaction {
    Connection getConnection() throws SQLException;
    void commit() throws SQLException;
    void rollback() throws SQLException;
    void close() throws SQLException;
}
```

**MyBatis官方Transaction接口**：
```java
public interface Transaction {
    Connection getConnection() throws SQLException;
    void commit() throws SQLException;
    void rollback() throws SQLException;
    void close() throws SQLException;
    Integer getTimeout() throws SQLException;
    void reset() throws SQLException;
    TransactionType getType();
}
```

### 5.2 JdbcTransaction实现

**LightSSM实现**：
```java
public class JdbcTransaction implements Transaction {
    private Connection connection;
    private DataSource dataSource;
    private boolean autoCommit = true;
    
    @Override
    public Connection getConnection() throws SQLException {
        if (connection == null) {
            connection = dataSource.getConnection();
            connection.setAutoCommit(autoCommit);
        }
        return connection;
    }
    
    @Override
    public void commit() throws SQLException {
        if (connection != null && !autoCommit) {
            connection.commit();
        }
    }
    
    @Override
    public void rollback() throws SQLException {
        if (connection != null && !autoCommit) {
            connection.rollback();
        }
    }
    
    @Override
    public void close() throws SQLException {
        if (connection != null) {
            connection.close();
        }
    }
}
```

### 5.3 与MyBatis官方对比

**MyBatis官方的JdbcTransaction**：
- 支持配置事务超时
- 支持保存点（Savepoint）
- 支持事务隔离级别配置
- 支持从连接池获取连接时保持原autoCommit状态

**LightSSM简化**：
- 基础的commit/rollback/close
- 简单的autoCommit控制
- 不支持保存点
- 不支持超时

```java
public class JdbcTransactionFactory implements TransactionFactory {
    @Override
    public void setProperties(Properties props) {
        // 可以配置事务相关属性
```

## 7. 连接池状态监控

### 7.1 PoolState状态管理

**LightSSM实现**：
```java
public class PoolState {
    protected PooledDataSource dataSource;
    
    public int getActiveConnectionCount() {
        synchronized (dataSource) {
            return dataSource.activeConnections.size();
        }
    }
    
    public int getIdleConnectionCount() {
        synchronized (dataSource) {
            return dataSource.idleConnections.size();
        }
    }
    
    public int getTotalConnectionCount() {
        return getActiveConnectionCount() + getIdleConnectionCount();
    }
}
```

**MyBatis官方PoolState**还包含：
- 累计请求次数
- 累计错误次数
- 平均等待时间
- 平均检出时间

## 8. 设计模式总结

### 8.1 工厂模式

```java
// 数据源工厂
public interface DataSourceFactory {
    DataSource getDataSource();
}

// 事务工厂
public interface TransactionFactory {
    Transaction newTransaction(DataSource ds);
}
```

### 8.2 代理模式

```java
// PooledConnection代理
public class PooledConnection implements InvocationHandler {
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        if ("close".equals(method.getName())) {
            returnToPool(); // 归还连接池
            return null;
        }
        return method.invoke(realConnection, args);
    }
}
```

### 8.3 单例模式

```java
// 每种数据源工厂都是单例使用
@Configuration
public class DataSourceConfig {
    @Bean
    public DataSource dataSource() {
        PooledDataSourceFactory factory = new PooledDataSourceFactory();
        factory.setProperties(props);
        return factory.getDataSource(); // 单例Bean
    }
}
```

## 9. 面试考点

### 9.1 高频问题

1. **数据库连接池的作用是什么？**
   - 减少连接创建和销毁的开销
   - 控制并发连接数
   - 提高系统响应速度

2. **常见的连接池有哪些？**
   - DBCP（Apache）
   - C3P0
   - Druid（阿里）
   - HikariCP（Spring Boot默认）

3. **事务的ACID特性是什么？**
   - 原子性（Atomicity）
   - 一致性（Consistency）
   - 隔离性（Isolation）
   - 持久性（Durability）

### 9.2 深度问题

1. **连接池如何避免连接泄露？**
   - 设置最大检出时间
   - 定期检测空闲连接
   - 使用try-with-resources

2. **HikariCP为什么性能最好？**
   - 字节码级别优化
   - 无锁设计
   - 连接快速检测

## 10. 总结

LightSSM的数据源与事务模块实现了：

- **三种数据源**：Unpooled、Pooled、Druid
- **连接池管理**：空闲/活跃列表
- **事务管理**：JdbcTransaction
- **工厂模式**：DataSourceFactory/TransactionFactory

通过对照MyBatis官方源码，可以看到LightSSM保留了数据源与事务的核心机制，剥离了高级特性（如连接检测、统计监控等），非常适合学习理解。
