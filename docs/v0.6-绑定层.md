# v0.6 绑定层

## 版本信息
- **版本号**: v0.6
- **提交哈希**: 00b9984

## 模块概述

绑定层负责将Mapper接口（DAO）绑定到具体的SQL实现，是ORM框架实现"接口编程"的关键。本模块包含：

1. **MapperProxy** - JDK动态代理
2. **MapperMethod** - 方法调用处理
3. **MapperRegistry** - 映射器注册中心
4. **MapperProxyFactory** - 代理工厂

---

## 1. 核心原理

### 1.1 为什么需要绑定层？

传统JDBC：
```java
// 直接写SQL
User user = jdbc.query("SELECT * FROM user WHERE id = ?", 1);
```

MyBatis方式：
```java
// 接口调用，自动执行对应SQL
UserDao dao = session.getMapper(UserDao.class);
User user = dao.findById(1);  // 自动执行映射的SQL
```

### 1.2 绑定流程

```
UserDao dao = session.getMapper(UserDao.class);
        ↓
MapperProxyFactory.newInstance(session)
        ↓
JDK动态代理 → MapperProxy
        ↓
user.findById(1) 方法调用
        ↓
MapperProxy.invoke()
        ↓
MapperMethod.execute()
        ↓
SqlSession.selectOne()
        ↓
Executor.query() → ...
```

## 2. MapperProxy - 代理类

### 2.1 实现原理

```java
public class MapperProxy<T> implements InvocationHandler, Serializable {
    private SqlSession sqlSession;
    private final Class<T> mapperInterface;
    private final Map<Method, MapperMethod> methodCache;
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) {
        if (Object.class.equals(method.getDeclaringClass())) {
            // toString(), hashCode() 等Object方法直接调用
            return method.invoke(this, args);
        } else {
            // DAO方法，走MapperMethod处理
            final MapperMethod mapperMethod = cachedMapperMethod(method);
            return mapperMethod.execute(sqlSession, args);
        }
    }
}
```

### 2.2 代理机制

```java
// invoke() 方法调用流程
proxy.findUser(1)
       ↓
InvocationHandler.invoke(proxy, method, [1])
       ↓
MapperProxy.invoke(proxy, findUser, [1])
       ↓
MapperMethod.execute(sqlSession, [1])
       ↓
sqlSession.selectOne("com.rongx.UserDao.findUser", 1)
```

### 2.3 方法缓存

```java
private MapperMethod cachedMapperMethod(Method method) {
    // 1. 先从缓存获取
    MapperMethod mapperMethod = methodCache.get(method);
    if (mapperMethod == null) {
        // 2. 缓存没有则创建
        mapperMethod = new MapperMethod(mapperInterface, method, 
            sqlSession.getConfiguration());
        // 3. 加入缓存
        methodCache.put(method, mapperMethod);
    }
    return mapperMethod;
}
```

## 3. MapperMethod - 方法调用处理

### 3.1 组成

```java
public class MapperMethod {
    private final SqlCommand command;     // SQL命令
    private final MethodSignature method; // 方法签名
}
```

### 3.2 SqlCommand - SQL命令

```java
public static class SqlCommand {
    private final String name;     // MappedStatement ID
    private final SqlCommandType type;  // SQL类型
    
    public SqlCommand(Configuration config, Class<?> mapperInterface, Method method) {
        // 获取statement ID: com.rongx.UserDao.findById
        String statementName = mapperInterface.getName() + "." + method.getName();
        MappedStatement ms = config.getMappedStatement(statementName);
        
        name = ms.getId();
        type = ms.getSqlCommandType();
    }
}
```

### 3.3 MethodSignature - 方法签名

```java
public static class MethodSignature {
    private final boolean returnsMany;  // 是否返回集合
    private final Class<?> returnType;  // 返回类型
    private final SortedMap<Integer, String> params;  // 参数索引
    
    public MethodSignature(Configuration config, Method method) {
        this.returnType = method.getReturnType();
        this.returnsMany = isCollectionOrArray(returnType);
        this.params = getParams(method);
    }
}
```

### 3.4 参数转换

```java
public Object convertArgsToSqlCommandParam(Object[] args) {
    // 1. 无参数
    if (args == null || params.size() == 0) {
        return null;
    }
    // 2. 单参数直接返回
    if (params.size() == 1) {
        return args[params.firstKey()];
    }
    // 3. 多参数转为Map
    Map<String, Object> param = new ParamMap<>();
    int i = 0;
    for (Map.Entry<Integer, String> entry : params.entrySet()) {
        // 支持 #{0}, #{1} 索引访问
        param.put(String.valueOf(entry.getKey()), args[entry.getKey()]);
        // 支持 #{param1}, #{param2} 名称访问
        param.put("param" + (i + 1), args[entry.getKey()]);
        i++;
    }
    return param;
}
```

### 3.5 执行方法

```java
public Object execute(SqlSession sqlSession, Object[] args) {
    Object param = method.convertArgsToSqlCommandParam(args);
    
    switch (command.getType()) {
        case SELECT:
            // 返回集合还是单个
            if (method.returnsMany) {
                return sqlSession.selectList(command.getName(), param);
            } else {
                return sqlSession.selectOne(command.getName(), param);
            }
        case INSERT:
            return sqlSession.insert(command.getName(), param);
        case UPDATE:
            return sqlSession.update(command.getName(), param);
        case DELETE:
            return sqlSession.delete(command.getName(), param);
        default:
            throw new RuntimeException("Unknown execution method");
    }
}
```

## 4. MapperRegistry - 映射器注册

### 4.1 核心功能

```java
public class MapperRegistry {
    private Configuration config;
    private final Map<Class<?>, MapperProxyFactory<?>> knownMappers = new HashMap<>();
    
    // 注册Mapper
    public <T> void addMapper(Class<T> type) {
        // 必须是接口
        if (!type.isInterface()) {
            throw new RuntimeException("Mapper must be interface");
        }
        // 注册代理工厂
        knownMappers.put(type, new MapperProxyFactory<>(type));
        
        // 解析注解配置
        MapperAnnotationBuilder parser = new MapperAnnotationBuilder(config, type);
        parser.parse();
    }
    
    // 获取Mapper代理
    public <T> T getMapper(Class<T> type, SqlSession sqlSession) {
        MapperProxyFactory<T> factory = knownMappers.get(type);
        return factory.newInstance(sqlSession);
    }
}
```

### 4.2 注册流程

```
configuration.addMapper(UserDao.class)
       ↓
MapperRegistry.addMapper(UserDao.class)
       ↓
knownMappers.put(UserDao.class, new MapperProxyFactory<>())
       ↓
MapperAnnotationBuilder.parse()  解析注解
       ↓
MappedStatement → Configuration
```

### 4.3 注解解析

```java
public class MapperAnnotationBuilder {
    public void parse() {
        // 遍历所有方法
        for (Method method : mapperInterface.getMethods()) {
            // 检查@Select注解
            Select select = method.getAnnotation(Select.class);
            if (select != null) {
                // 解析SQL并注册
                parseSelect(method, select.value());
            }
        }
    }
}
```

## 5. MapperProxyFactory - 代理工厂

### 5.1 工厂方法

```java
public class MapperProxyFactory<T> {
    private final Class<T> mapperInterface;
    
    public T newInstance(SqlSession sqlSession) {
        // 创建MapperProxy
        MapperProxy<T> proxy = new MapperProxy<>(sqlSession, mapperInterface, methodCache);
        // 创建代理对象
        return (T) Proxy.newProxyInstance(
            mapperInterface.getClassLoader(),
            new Class[]{mapperInterface},
            proxy
        );
    }
}
```

### 5.2 创建过程

```java
// 1. 创建Handler
MapperProxy<T> proxy = new MapperProxy<>(
    sqlSession,           // SqlSession
    mapperInterface,     // 接口Class
    new HashMap<>()      // 方法缓存
);

// 2. 创建代理对象
return (T) Proxy.newProxyInstance(
    mapperInterface.getClassLoader(),
    new Class[]{mapperInterface},  // 代理的接口
    proxy                       // InvocationHandler
);
```

## 6. 完整调用流程

### 6.1 获取Mapper

```java
SqlSession session = factory.openSession();
UserDao dao = session.getMapper(UserDao.class);
```

流程：
```
session.getMapper(UserDao.class)
       ↓
Configuration.getMapper(UserDao.class, session)
       ↓
MapperRegistry.getMapper(UserDao.class, session)
       ↓
MapperProxyFactory.newInstance(session)
       ↓
new MapperProxy<>(session, UserDao.class, cache)
       ↓
Proxy.newProxyInstance(..., proxy)
       ↓
UserDao dao (代理对象)
```

### 6.2 调用方法

```java
User user = dao.findById(1);
```

流程：
```
dao.findById(1)
       ↓
Proxy.invoke(proxy, findById, [1])
       ↓
MapperProxy.invoke()
       ├─→ cachedMapperMethod(findById)
       │        ↓
       │    new MapperMethod(mapperInterface, method, config)
       │        ↓
       │    methodCache.put(method, mapperMethod)
       │
       └─→ mapperMethod.execute(sqlSession, [1])
                ├─→ convertArgsToSqlCommandParam([1])
                ├─→ sqlSession.selectOne("com.rongx.UserDao.findById", param)
                └─→ 返回User对象
```

## 7. 多参数支持

Map<String, Object> param = new ParamMap<>();

### 7.2 SQL中使用

```xml
<select id="findByIdAndName" resultType="User">
    SELECT * FROM user WHERE id = #{0} AND name = #{param2}
</select>
```

## 10. 线程安全

### 10.1 SqlSession

```java
// 非线程安全，每个线程独立
SqlSession session = factory.openSession();
UserDao dao = session.getMapper(UserDao.class);
```

### 10.2 MapperProxy

```java
// 线程安全，可以共享
UserDao dao = session.getMapper(UserDao.class);
// 多个线程可以同时调用
```

**原因**：MapperProxy内部只引用SqlSession，无状态。

## 11. 面试问题

### Q1: MapperProxy的invoke方法何时被调用？
**Answer**: 当调用Mapper接口方法时，JDK动态代理会触发invoke方法。

### Q2: methodCache的作用？
**Answer**: 缓存MapperMethod，避免每次调用都重新创建，提高性能。

### Q3: 多参数如何处理？
**Answer**: 转换为ParamMap，支持#{0}、#{param1}等多种访问方式。

### Q4: getMapper的实现原理？
**Answer**: 使用JDK动态代理，创建MapperProxy作为InvocationHandler。

## 12. 总结

绑定层要点：
1. **JDK动态代理**实现接口调用
2. **MapperProxy**拦截方法调用
3. **MapperMethod**解析方法并执行SQL
4. **MapperRegistry**管理Mapper注册

这是ORM框架实现"接口编程"的核心，用户只需定义接口和注解，无需编写实现类。
