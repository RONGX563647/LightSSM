## 1. 反射模块 (reflection)

### 1.1 MetaClass - 元类

MetaClass封装类的元信息，提供属性getter/setter操作。

```java
public class MetaClass {
    private Class<?> type;
    private Reflector reflector;
    
    // 创建MetaClass
    public static MetaClass forClass(Class<?> type) {
        return new MetaClass(Reflector.getReflector(type));
    }
    
    // 获取setter类型
    public Class<?> getSetterType(String property) {
        return reflector.getSetterType(property);
    }
    
    // 获取getter类型
    public Class<?> getGetterType(String property) {
        return reflector.getGetterType(property);
    }
    
    // 是否有setter
    public boolean hasSetter(String property) {
        return reflector.hasSetter(property);
    }
    
    // 是否有getter
    public boolean hasGetter(String property) {
        return reflector.hasGetter(property);
    }
}
```

### 1.2 Reflector - 反射器

Reflector封装单个类的反射信息。

```java
public class Reflector {
    private Class<?> type;
    private Method[] getters;  // getter方法
    private Method[] setters; // setter方法
    
    // 构造时解析
    public Reflector(Class<?> clazz) {
        this.type = clazz;
        addGetters(clazz.getDeclaredMethods());
        addSetters(clazz.getDeclaredMethods());
    }
}
```

### 1.3 MetaObject - 元对象

MetaObject是对象的包装器，通过它可以操作对象属性。

```java
public class MetaObject {
    private Object object;
    private ObjectFactory objectFactory;
    private ObjectWrapper objectWrapper;
    
    // 设置属性值
    public void setValue(String property, Object value) {
        // 1. 解析属性名
        PropertyTokenizer tokenizer = new PropertyTokenizer(property);
        if (tokenizer.hasNext()) {
            // 2. 嵌套属性
            MetaObject metaValue = metaObject.findProperty(tokenizer.getName());
            metaValue.setValue(tokenizer.getChildren(), value);
        } else {
            // 3. 直接设置
            objectWrapper.setValue(tokenizer.getName(), value);
        }
    }
    
    // 获取属性值
    public Object getValue(String property) {
        // 类似逻辑
    }
}
```

### 1.4 PropertyTokenizer - 属性分词器

解析嵌套属性，如 "user.address.city"。

```java
public class PropertyTokenizer(String expression) {
    private String name;        // 当前属性名
    private String children;   // 剩余属性
    
    public PropertyTokenizer(String expression) {
        // user.address.city
        //         ↓
        // name = user
        // children = address.city
    }
}
```

### 1.5 使用场景

**结果映射**：
```java
MetaObject metaObject = configuration.newMetaObject(user);
metaObject.setValue("userName", "Tom");  // 设置属性
String name = (String) metaObject.getValue("userName");
```

## 2. 插件模块 (plugin)

### 2.1 Interceptor - 拦截器接口

```java
public interface Interceptor {
    // 拦截方法
    Object intercept(Invocation invocation) throws Throwable;
    
    // 包装目标对象
    Object plugin(Object target);
    
    // 设置属性
    void setProperties(Properties properties);
}
```

### 2.2 Invocation - 调用封装

```java
public class Invocation {
    private Object target;      // 目标对象
    private Method method;   // 方法
    private Object[] args;    // 参数
    
    public Object proceed() throws Throwable {
        return method.invoke(target, args);
    }
}
```

### 2.3 InterceptorChain - 拦截器链

```java
public class InterceptorChain {
    private final List<Interceptor> interceptors = new ArrayList<>();
    
    // 包装目标对象
    public Object pluginAll(Object target) {
        for (Interceptor interceptor : interceptors) {
            target = interceptor.plugin(target);
        }
        return target;
    }
}
```

### 2.4 插件机制原理

```
Executor → plugin(Executor)
           ↓
Plugin.wrap(executor, interceptor)
           ↓
JDK/CGLIB代理
           ↓
intercept() → invocation.proceed()
```

### 2.5 分页插件示例

```java
@Intercepts({
    @Signature(type = Executor.class, method = "query", args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class, BoundSql.class})
})
public class PageInterceptor implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        // 1. 获取参数
        Object[] args = invocation.getArgs();
        RowBounds rowBounds = (RowBounds) args[2];
        
        // 2. 拦截查询
        if (rowBounds != RowBounds.DEFAULT) {
            // 处理分页
            args[2] = new RowBounds(0, 10);  // 第一页10条
        }
        
        // 3. 继续执行
        return invocation.proceed();
    }
}
```

### 2.6 配置插件

```xml
<plugins>
    <plugin interceptor="com.rongx.plugin.PageInterceptor">
        <property name="dialect" value="mysql"/>
    </plugin>
</plugins>
```

### 2.7 设计模式

| 模式 | 原理 |
|------|------|
| 责任链 | 多个拦截器串连 |
| 装饰器 | 层层包装 |
| 代理 | 增强目标对象 |

## 3. 脚本模块 (scripting)

### 3.1 动态SQL概述

动态SQL支持条件判断、循环等逻辑：

```xml
<select id="findUser">
    SELECT * FROM user
    <where>
        <if test="id != null">
            AND id = #{id}
        </if>
        <if test="name != null">
            AND name = #{name}
        </if>
    </where>
</select>
```

### 3.2 SqlNode - SQL节点接口

```java
public interface SqlNode {
    // 应用节点逻辑
    boolean apply(DynamicContext context);
}
```

### 3.3 IfSqlNode - IF节点

```java
public class IfSqlNode implements SqlNode {
    private SqlNode contents;
    private String test;
    
    @Override
    public boolean apply(DynamicContext context) {
        // 1. 解析OGNL表达式
        if (OgnlCache.evaluateBoolean(test, context.getBindings())) {
            // 2. 条件成立，应用子节点
            return contents.apply(context);
        }
        return false;
    }
}
```

### 3.4 WhereSqlNode - WHERE节点

```java
public class WhereSqlNode implements SqlNode {
    private List<SqlNode> contents;
    
    @Override
    public boolean apply(DynamicContext context) {
        StringBuilder sql = new StringBuilder();
        for (SqlNode content : contents) {
            content.apply(context);
            // 处理AND/OR
        }
        // 移除多余的 AND/WHERE
        return true;
    }
}
```

### 3.5 ForeachSqlNode - 循环节点

```java
public class ForeachSqlNode implements SqlNode {
    private String collection;
    private String item;
    private String open;
    private String close;
    private String separator;
    
    @Override
    public boolean apply(DynamicContext context) {
        // 遍历集合
        Collection<?> values = (Collection<?>) OgnlCache.evaluate(collection, context.getBindings());
        for (Object value : values) {
            // 生成 #{item}
            context.append(item);
            // 处理分隔符
        }
        return true;
    }
}
```

### 3.6 OGNL表达式

OGNL(Object Graph Navigation Language)用于动态SQL中的表达式：

| 表达式 | 说明 |
|--------|------|
| `#{id != null}` | 非空判断 |
| `#{name == 'Tom'}` | 字符串判断 |
| `#{list.size() > 0}` | 集合判断 |
| `#{user.name}` | 嵌套属性 |

### 3.7 DynamicSqlSource - 动态SQL源

```java
public class DynamicSqlSource implements SqlSource {
    private SqlNode root;
    
    @Override
    public BoundSql getBoundSql(Object parameterObject) {
        DynamicContext context = new DynamicContext(parameterObject);
        // 1. 应用所有节点
        root.apply(context);
        // 2. 生成SQL
        return new BoundSql(context.getSql(), ...);
    }
}
```

### 3.8 XMLScriptBuilder - 脚本构建器

```java
public class XMLScriptBuilder extends BaseBuilder {
    public SqlNode parseScriptNode(Element element) {
        List<SqlNode> contents = new ArrayList<>();
        // 解析if/where/foreach等节点
        for (Element child : element.elements()) {
            String name = child.getName();
            if ("if".equals(name)) {
                contents.add(new IfSqlNode(...));
            } else if ("where".equals(name)) {
                contents.add(parseWhereNode(child));
            }
        }
        return new MixedSqlNode(contents);
    }
}
```

## 4. 核心流程图

### 4.1 动态SQL处理流程

```
SELECT * FROM user
<where>
    <if test="id != null">AND id = #{id}</if>
</where>
           ↓
XMLScriptBuilder.parseScriptNode()
           ↓
创建IfSqlNode + WhereSqlNode
           ↓
DynamicSqlSource.getBoundSql(parameter)
           ↓
root.apply(context)
           ↓
生成: SELECT * FROM user WHERE id = ?
```

### 4.2 插件执行流程

```
Executor query()
       ↓
pluginAll(Executor)
       ↓
interceptor1.plugin(executor)
       ↓
interceptor2.plugin(proxy)
       ↓
实际查询时
       ↓
invoke() → intercept()
       ↓
invocation.proceed()
```

## 6. 面试问题

**Answer**: 对象的属性getter/setter操作，支持嵌套属性如user.address.city。

### Q2: 插件如何拦截方法？
**Answer**: 通过拦截器包装目标对象，调用intercept方法。

**Answer**: SqlNode树形结构，OGNL表达式判断，动态生成SQL。
