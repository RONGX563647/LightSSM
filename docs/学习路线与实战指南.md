## 三、实战练习

### 3.1 练习1: 添加一级缓存

**目标**：实现SqlSession级缓存，减少重复查询

**步骤**：

```java
// 1. 在BaseExecutor中添加缓存字段
private final Map<String, CacheObject> localCache = new HashMap<>();

// 2. 修改query方法
@Override
public <E> List<E> query(...) {
    // 生成缓存key
    String cacheKey = ms.getId() + ":" + parameter;
    
    // 查询缓存
    if (localCache.containsKey(cacheKey)) {
        // 从缓存获取
        return (List<E>) localCache.get(cacheKey);
    }
    
    // 没有缓存，执行查询
    List<E> result = doQuery(...);
    
    // 放入缓存
    localCache.put(cacheKey, result);
    
    return result;
}

// 3. 添加clearLocalCache()方法
public void clearLocalCache() {
    localCache.clear();
}
```

### 3.2 练习2: 自定义TypeHandler

**目标**：处理自定义枚举类型

```java
// 1. 定义枚举
public enum Status {
    ACTIVE,
    INACTIVE,
    PENDING
}

// 2. 创建TypeHandler
public class StatusTypeHandler extends BaseTypeHandler<Status> {
    
    @Override
    protected void setNonNullParameter(PreparedStatement ps, int i, 
                                Status parameter, JdbcType jdbcType) {
        // 枚举 → 字符串
        ps.setString(i, parameter.name());
    }
    
    @Override
    protected Status getNullableResult(ResultSet rs, String columnName) {
        String value = rs.getString(columnName);
        return value == null ? null : Status.valueOf(value);
    }
    
    @Override
    public Status getNullableResult(ResultSet rs, int columnIndex) {
        String value = rs.getString(columnIndex);
        return value == null ? null : Status.valueOf(value);
    }
}

// 3. 注册使用
// 方式1: XML配置
<typeHandlers>
    <package name="com.rongx.handler"/>
</typeHandlers>

// 方式2: 注解
@Results({
    @Result(property="status", column="status", 
           typeHandler=StatusTypeHandler.class)
})
```

### 3.3 练习3: 编写分页插件

```java
@Intercepts({
    @Signature(type = Executor.class, method = "query", 
               args = {MappedStatement.class, Object.class, RowBounds.class, 
                     ResultHandler.class, BoundSql.class})
})
public class PageInterceptor implements Interceptor {
    
    private static final ThreadLocal<Page> PAGE = new ThreadLocal<>();
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        Object[] args = invocation.getArgs();
        RowBounds rowBounds = (RowBounds) args[2];
        
        // 判断是否需要分页
        if (rowBounds instanceof Page) {
            Page page = (Page) rowBounds;
            
            // 获取BoundSql
            BoundSql boundSql = (BoundSql) args[4];
            
            // 添加分页SQL
            String sql = boundSql.getSql();
            sql = sql + " LIMIT " + page.getOffset() + ", " + page.getLimit();
            
            // 修改SQL (通过反射)
            Field sqlField = BoundSql.class.getDeclaredField("sql");
            sqlField.setAccessible(true);
            sqlField.set(boundSql, sql);
            
            // 记录总数
            PAGE.set(page);
        }
        
        return invocation.proceed();
    }
    
    @Override
    public Object plugin(Object target) {
        return Plugin.wrap(target, this);
    }
}
```

### 3.4 练习4: 实现延迟加载

```java
// 1. 创建LazyLoader
public class LazyLoader {
    private Object target;
    private String property;
    private MetaObject metaObject;
    
    public Object load() {
        // 查询关联对象
        Object value = metaObject.getValue(property);
        
        // 清空占位符，表示已加载
        metaObject.setValue(property, null);
        
        return value;
    }
}

// 2. 修改ResultSetHandler
private Object getRowValue(ResultSetWrapper rsw, ResultMap rm) {
    Object resultObject = createResultObject(...);
    
    // 判断是否有延迟加载
    if (rm.hasLazyAssociations()) {
        MetaObject metaObject = configuration.newMetaObject(resultObject);
        
        // 为每个关联属性创建代理
        for (Association assoc : rm.getAssociations()) {
            LazyLoader loader = new LazyLoader(resultObject, assoc.getProperty(), metaObject);
            
            // 创建代理对象
            Object proxy = Proxy.newProxyInstance(
                assoc.getJavaType().getClassLoader(),
                new Class[]{assoc.getJavaType()},
                (p, m, a) -> loader.load()
            );
            
            metaObject.setValue(assoc.getProperty(), proxy);
        }
    }
    
    return resultObject;
}
```

## 四、常见问题与解决方案

### 4.1 性能问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 反射频繁 | 每次都反射 | 缓存Method |
| OGNL重复解析 | 每次都解析 | 缓存表达式 |
| 查询重复 | 无缓存 | 添加一级缓存 |
| 连接创建 | 无连接池 | 使用连接池 |

### 4.2 功能问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 关联对象无法加载 | 未实现association | 实现嵌套查询 |
| 集合无法映射 | 未实现collection | 实现集合映射 |
| 延迟加载失败 | 未实现LazyLoader | 实现延迟加载 |

### 4.3 兼容性问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Oracle不支持自增 | Oracle无AUTO_INCREMENT | 使用序列 |
| 列名大小写 | 数据库大小写敏感 | 配置mapUnderscoreToCamelCase |
| 字符集 | 编码不一致 | 配置charset |
