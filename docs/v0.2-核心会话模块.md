## 版本信息
- **版本号**: v0.2

## 模块概述

核心会话模块是整个ORM框架的门面和入口，提供了统一的数据操作API。本模块包含：

1. **SqlSession** - 会话接口，CRUD操作入口
2. **SqlSessionFactory** - 会话工厂
3. **SqlSessionFactoryBuilder** - 工厂构建器
4. **Configuration** - 全局配置中心

---

## 1. SqlSession - 核心API

### 1.1 接口定义

```java
public interface SqlSession extends Closeable {
    <T> T selectOne(String statement, Object parameter);
    <E> List<E> selectList(String statement, Object parameter);
    int insert(String statement, Object parameter);
    int update(String statement, Object parameter);
    Object delete(String statement, Object parameter);
    
    void commit();
    Configuration getConfiguration();
    <T> T getMapper(Class<T> type);
    
    @Override
    default void close() { }
}
```

### 1.2 方法分类

| 方法 | 说明 |
|------|------|
| selectOne | 查询单条 |
| selectList | 查询多条 |
| insert | 插入 |
| update | 更新 |
| delete | 删除 |
| commit | 提交事务 |
| getMapper | 获取Mapper代理 |

### 1.3 核心方法详解

#### selectOne 方法

```java
<T> T selectOne(String statement, Object parameter);
```

**执行流程**：
```
selectOne("getUser", userId)
       ↓
1. 获取MappedStatement (statement + namespace)
       ↓
2. 创建Executor
       ↓
3. Executor.query()
       ↓
4. StatementHandler.query()
       ↓
5. PreparedStatement.execute()
       ↓
6. ResultSetHandler.handleResult()
       ↓
7. 返回结果对象
```

#### selectList 方法

```java
<E> List<E> selectList(String statement, Object parameter);
```

**与selectOne的区别**：
- 返回类型是List而非单个对象
- 内部使用ResultHandler收集结果
- 支持RowBounds分页

#### getMapper 方法

```java
<T> T getMapper(Class<T> type);
```

**核心原理**：
```
UserDao dao = session.getMapper(UserDao.class);
       ↓
1. Configuration.getMapper(type, session)
       ↓
2. MapperRegistry.getMapper(type, session)
       ↓
3. MapperProxyFactory.newInstance(session)
       ↓
4. new MapperProxy<>(session)
       ↓
5. 返回JDK动态代理对象
```

### 1.4 事务管理

```java
void commit();  // 提交事务
```

**底层实现**：
```java
// DefaultSqlSession中
public void commit() {
    transaction.commit();  // 底层调用 JDBC connection.commit()
}
```

## 4. Configuration - 全局配置中心

### 4.1 核心职责

Configuration是整个框架的"大管家"，集中管理所有配置：

```java
public class Configuration {
    // 环境配置
    protected Environment environment;
    
    // 映射注册中心
    protected MapperRegistry mapperRegistry;
    
    // 映射语句存储
    protected Map<String, MappedStatement> mappedStatements;
    
    // 结果映射存储
    protected Map<String, ResultMap> resultMaps;
    
    // 类型处理器注册
    protected TypeHandlerRegistry typeHandlerRegistry;
    
    // 插件链
    protected InterceptorChain interceptorChain;
    
    // 对象工厂
    protected ObjectFactory objectFactory;
}
```

### 4.2 组件创建工厂

Configuration不仅存储配置，还负责创建各种组件：

```java
// 创建执行器
public Executor newExecutor(Transaction transaction) {
    return new SimpleExecutor(this, transaction);
}

// 创建语句处理器
public StatementHandler newStatementHandler(...) {
    return new PreparedStatementHandler(...);
}

// 创建结果集处理器
public ResultSetHandler newResultSetHandler(...) {
    return new DefaultResultSetHandler(...);
}

// 创建参数处理器
public ParameterHandler newParameterHandler(...) {
    return mappedStatement.getLang()
        .createParameterHandler(...);
}
```

### 4.3 存储结构图

```
Configuration
├── environment          → 环境配置
├── mapperRegistry       → Mapper注册表
├── mappedStatements    → SQL映射集合 (Map<String, MappedStatement>)
├── resultMaps          → 结果映射集合
├── typeHandlerRegistry  → 类型处理器
├── interceptorChain    → 插件链
├── objectFactory       → 对象工厂
└── languageRegistry   → 脚本语言
```

## 5. 核心流程图

### 5.1 整体架构

```
用户代码
   ↓
SqlSession (门面)
   ↓
Configuration (配置中心)
   ├─→ Executor (执行器)
   │    └─→ StatementHandler
   │         ├─→ ParameterHandler
   │         └─→ ResultSetHandler
   ├─→ MapperRegistry
   │    └─→ MapperProxy
   └─→ TypeHandlerRegistry
```

### 5.2 查询流程

```
session.selectOne("findUser", 1)
         ↓
DefaultSqlSession.selectOne()
         ↓
Executor.query()
         ↓
StatementHandler.query()
         ├─→ ParameterHandler.setParameters()
         └─→ PreparedStatement.execute()
         ↓
ResultSetHandler.handleResultSets()
         ↓
结果对象
```

### 5.3 Mapper获取流程

```
session.getMapper(UserDao.class)
         ↓
Configuration.getMapper(UserDao.class, session)
         ↓
MapperRegistry.getMapper(UserDao.class, session)
         ↓
MapperProxyFactory.newInstance(session)
         ↓
JDK动态代理
         ↓
UserDao dao (代理对象)
```

## 7. 线程安全分析

### 7.1 SqlSession

```java
// 非线程安全，每个线程独立创建
SqlSession session = factory.openSession();
// 使用后必须关闭
session.close();
```

**为什么非线程安全？**
- 内部持有Connection
- 缓存状态（Executor等）

### 7.2 SqlSessionFactory

```java
// 线程安全，可以共享
SqlSessionFactory factory = ...;
```

**为什么线程安全？**
- 只读配置
- 无状态

### 7.3 Configuration

```java
// 线程安全，只读配置
Configuration config = ...;
```

**为什么线程安全？**
- 设计为只读
- 添加同步控制

## 8. 生命周期管理

### 8.1 典型使用

```java
// 1. 创建工厂 (应用级，单例)
SqlSessionFactory factory = new SqlSessionFactoryBuilder()
    .build(reader);

// 2. 获取会话 (方法级)
try (SqlSession session = factory.openSession()) {
    // 3. 操作
    User user = session.selectOne("findUser", 1);
    session.commit();
} // 自动关闭
```

### 8.2 生命周期图

```
应用级别: SqlSessionFactory (单例，整个应用一个)
         ↓
方法级别: SqlSession (每个线程独立)
         ↓
请求级别: Transaction, Connection
```

## 9. 面试问题

### Q1: SqlSession和Connection的关系？
**Answer**: SqlSession内部持有Connection，一个SqlSession对应一个Connection。

### Q2: 为什么SqlSession不是线程安全的？
**Answer**: SqlSession持有数据库连接和缓存状态，多线程共享会导致数据竞争。

### Q3: Configuration的作用？
**Answer**: 全局配置中心，管理MappedStatement、TypeHandler、Executor等所有组件。

### Q4: getMapper的原理？
**Answer**: 使用JDK动态代理，MapperProxy拦截方法调用，转换为SqlSession调用。
