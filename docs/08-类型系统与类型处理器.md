## 1. 类型系统架构

### 1.1 TypeHandler接口

**LightSSM实现**：
```java
public interface TypeHandler<T> {

    T getResult(ResultSet rs, String columnName) throws SQLException;
```

## 2. 基础类型处理器

### 2.1 BaseTypeHandler基类

**LightSSM实现**：
```java
public abstract class BaseTypeHandler<T> implements TypeHandler<T> {
    
    @Override
    public void setParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) throws SQLException {
        if (parameter == null) {
            if (jdbcType == null) {
                throw new TypeException("JDBC type is null");
            }
            ps.setNull(i, jdbcType.TYPE_CODE);
        } else {
            setNonNullParameter(ps, i, parameter, jdbcType);
        }
    }
    
    protected abstract void setNonNullParameter(PreparedStatement ps, int i, T parameter, JdbcType jdbcType) throws SQLException;
}
```

## 3. 类型处理器注册表

### 3.1 TypeHandlerRegistry

**LightSSM实现**：
```java
public class TypeHandlerRegistry {
    private final Map<JdbcType, TypeHandler<?>> jdbcTypeHandlerMap = new HashMap<>();
    private final Map<String, TypeHandler<?>> typeHandlerStringMap = new HashMap<>();
    private final Map<Class<?>, TypeHandler<?>> typeHandlerClassMap = new HashMap<>();
    
    public TypeHandlerRegistry() {
        // 注册内置类型处理器
        register(String.class, new StringTypeHandler());
        register(Integer.class, new IntegerTypeHandler());
        register(int.class, new IntegerTypeHandler());
        register(Long.class, new LongTypeHandler());
        register(long.class, new LongTypeHandler());
        register(Double.class, new DoubleTypeHandler());
        register(double.class, new DoubleTypeHandler());
        register(Boolean.class, new BooleanTypeHandler());
        register(boolean.class, new BooleanTypeHandler());
        register(Date.class, new DateTypeHandler());
        // ... 更多注册
    }
    
    @SuppressWarnings("unchecked")
    public <T> void register(Class<T> javaTypeClass, TypeHandler<?> handler) {
        if (javaTypeClass != null) {
            typeHandlerClassMap.put(javaTypeClass, handler);
        }
    }
    
    @SuppressWarnings("unchecked")
    public <T> TypeHandler<T> getTypeHandler(Class<T> type) {
        TypeHandler<?> handler = typeHandlerClassMap.get(type);
        if (handler == null) {
            // 查找父类
            for (Map.Entry<Class<?>, TypeHandler<?>> entry : typeHandlerClassMap.entrySet()) {
                if (entry.getKey().isAssignableFrom(type)) {
                    return (TypeHandler<T>) entry.getValue();
                }
            }
            // 默认返回Object类型处理器
            handler = typeHandlerClassMap.get(Object.class);
        }
        return (TypeHandler<T>) handler;
    }
}
```

### 3.2 与MyBatis官方对比

**MyBatis官方的TypeHandlerRegistry**：
- 支持类型超类查找
- 支持JDBC类型映射
- 支持类型处理器工厂
- 支持泛型类型处理器
- 支持TypeHandler包扫描
- 包含100+个内置TypeHandler

**LightSSM简化**：
- 基础的类型映射
- 20+个内置TypeHandler
- 不支持工厂和扫描

## 4. 类型别名系统

### 4.1 TypeAliasRegistry

**LightSSM实现**：
```java
public class TypeAliasRegistry {
    private final Map<String, Class<?>> typeAliases = new HashMap<>();
    
    public TypeAliasRegistry() {
        // 注册内置类型别名
        registerAlias("string", String.class);
        registerAlias("int", Integer.class);
        registerAlias("integer", Integer.class);
        registerAlias("long", Long.class);
        registerAlias("double", Double.class);
        registerAlias("boolean", Boolean.class);
        registerAlias("date", Date.class);
        registerAlias("map", Map.class);
        registerAlias("list", List.class);
    }
    
    public void registerAlias(String alias, Class<?> value) {
        if (alias == null) {
            throw new TypeException("Alias cannot be null");
        }
        typeAliases.put(alias.toLowerCase(Locale.ENGLISH), value);
    }
    
    public void registerAlias(String className) {
        try {
            registerAlias(className, Resources.classForName(className));
        } catch (ClassNotFoundException e) {
            throw new TypeException("Error registering type alias: " + className, e);
        }
    }
    
    public <T> Class<T> resolveAlias(String string) {
        if (string == null) return null;
        String key = string.toLowerCase(Locale.ENGLISH);
        Class<?> value = typeAliases.get(key);
        if (value == null) {
            try {
                return (Class<T>) Resources.classForName(string);
            } catch (ClassNotFoundException e) {
                throw new TypeException("Could not resolve type alias: " + string, e);
            }
        }
        return (Class<T>) value;
    }
}
```

### 4.2 与MyBatis官方对比

**MyBatis官方的TypeAliasRegistry**：
- 支持包扫描自动注册
- 支持内部类
- 支持泛型类
- 包含50+个内置别名
- 支持注解@Alias

**LightSSM简化**：
- 手动注册
- 基础类型别名
- 不支持包扫描

## 6. 类型转换场景

### 6.1 参数绑定

```java
// PreparedStatement参数绑定
public class DefaultParameterHandler implements ParameterHandler {
    @Override
    public void setParameters(PreparedStatement ps) throws SQLException {
        BoundSql boundSql = mappedStatement.getBoundSql();
        List<ParameterMapping> parameterMappings = boundSql.getParameterMappings();
        
        for (int i = 0; i < parameterMappings.size(); i++) {
            ParameterMapping parameterMapping = parameterMappings.get(i);
            Object value = boundSql.getParameterObject();
            JdbcType jdbcType = parameterMapping.getJdbcType();
            
            // 获取TypeHandler
            TypeHandler<?> typeHandler = parameterMapping.getTypeHandler();
            
            // 设置参数
            ((TypeHandler) typeHandler).setParameter(ps, i + 1, value, jdbcType);
        }
    }
}
```

### 6.2 结果映射

```java
// ResultSet结果映射
public class DefaultResultSetHandler implements ResultSetHandler {
    @Override
    public List<Object> handleResultSets(Statement stmt) throws SQLException {
        ResultSet rs = stmt.getResultSet();
        List<Object> resultObjects = new ArrayList<>();
        
        while (rs.next()) {
            Object resultObject = createResultObject(rs, resultMap);
            if (resultObject != null) {
                resultObjects.add(resultObject);
            }
        }
        
        return resultObjects;
    }
    
    private Object createResultObject(ResultSet rs, ResultMap resultMap) throws SQLException {
        Object resultObject = objectFactory.create(resultClass);
        
        for (ResultMapping resultMapping : resultMap.getResultMappings()) {
            String column = resultMapping.getColumn();
            String property = resultMapping.getProperty();
            Class<?> propertyType = resultMapping.getPropertyType();
            
            // 获取TypeHandler
            TypeHandler<?> typeHandler = resultMapping.getTypeHandler();
            
            // 获取结果值
            Object value = typeHandler.getResult(rs, column);
            
            // 设置属性
            MetaObject metaObject = configuration.newMetaObject(resultObject);
            metaObject.setValue(property, value);
        }
        
        return resultObject;
    }
}
```

## 7. 自定义类型处理器

### 7.1 实现方式

**LightSSM支持两种实现方式**：

方式一：继承BaseTypeHandler
```java
public class MyStringTypeHandler extends BaseTypeHandler<String> {
    @Override
    protected void setNonNullParameter(PreparedStatement ps, int i, String parameter, JdbcType jdbcType) throws SQLException {
        ps.setString(i, parameter.toUpperCase()); // 转为大写
    }
    
    @Override
    public String getResult(ResultSet rs, String columnName) throws SQLException {
        return rs.getString(columnName).toLowerCase(); // 转为小写
    }
    
    @Override
    public String getResult(ResultSet rs, int columnIndex) throws SQLException {
        return rs.getString(columnIndex).toLowerCase();
    }
}
```

方式二：实现TypeHandler接口
```java
public class MyIntegerTypeHandler implements TypeHandler<Integer> {
    @Override
    public void setParameter(PreparedStatement ps, int i, Integer parameter, JdbcType jdbcType) throws SQLException {
        ps.setInt(i, parameter * 100); // 放大100倍
    }
    
    @Override
    public Integer getResult(ResultSet rs, String columnName) throws SQLException {
        return rs.getInt(columnName) / 100; // 缩小100倍
    }
    
    @Override
    public Integer getResult(ResultSet rs, int columnIndex) throws SQLException {
        return rs.getInt(columnIndex) / 100;
    }
}
```

### 7.2 注册方式

**方式一：XML配置**
```xml
<configuration>
    <typeHandlers>
        <typeHandler handler="com.example.MyStringTypeHandler"/>
        <typeHandler javaType="java.lang.String" jdbcType="VARCHAR" 
                     handler="com.example.MyStringTypeHandler"/>
    </typeHandlers>
</configuration>
```

**方式二：注解配置**
```java
@MappedJdbcTypes(JdbcType.VARCHAR)
@MappedTypes(String.class)
public class MyStringTypeHandler extends BaseTypeHandler<String> {
    // ...
}
```

**方式三：代码注册**
```java
Configuration configuration = new Configuration();
configuration.getTypeHandlerRegistry().register(String.class, new MyStringTypeHandler());
```

## 9. 面试考点

### 9.1 高频问题

1. **TypeHandler的作用是什么？**
   - Java类型与JDBC类型的转换桥梁
   - 参数绑定和结果映射

2. **如何自定义TypeHandler？**
   - 继承BaseTypeHandler
   - 实现TypeHandler接口
   - 注册到TypeHandlerRegistry

3. **MyBatis有哪些内置TypeHandler？**
   - 基础类型：String/Integer/Long等
   - 日期类型：Date/Time/Timestamp
   - 大对象：Blob/Clob
   - 枚举：EnumTypeHandler

### 9.2 深度问题

1. **TypeHandler的查找顺序？**
   - 直接匹配javaType
   - 查找父类的TypeHandler
   - 使用默认的ObjectTypeHandler

2. **如何处理NULL值？**
   - setParameter时检查null
   - 使用setNull设置NULL
   - JdbcType必须指定

## 10. 总结

LightSSM的类型系统实现了：

- **TypeHandler接口**：类型转换策略
- **TypeHandlerRegistry**：类型处理器注册表
- **TypeAliasRegistry**：类型别名注册表
- **基础TypeHandler**：20+个内置处理器
- **JdbcType枚举**：20个JDBC类型

通过对照MyBatis官方源码，可以看到LightSSM保留了类型系统的核心机制，剥离了高级特性（如泛型处理、包扫描等），非常适合学习理解。
