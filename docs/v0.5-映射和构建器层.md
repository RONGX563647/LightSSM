## 版本信息
- **版本号**: v0.5

## 模块概述

映射层和构建器层负责SQL语句和结果映射的配置与解析，是ORM框架的配置核心。本模块包含：

1. **mapping** - 映射模型
2. **builder** - 构建器
3. **xml** - XML解析器
4. **annotation** - 注解构建器

---

## 1. 核心模型

### 1.1 MappedStatement - 映射语句

MappedStatement表示一个映射的SQL语句，存储在Configuration中。

```java
public class MappedStatement {
    private String id;                    // 语句ID (namespace + methodName)
    private SqlCommandType commandType; // SQL类型 (SELECT/INSERT/UPDATE/DELETE)
    private SqlSource sqlSource;       // SQL源 (静态/动态)
    private Class<?> resultType;        // 返回类型
    private List<ResultMap> resultMaps; // 结果映射
    private KeyGenerator keyGenerator;  // 主键生成器
    private LanguageDriver lang;       // 脚本语言
}
```

### 1.2 核心结构图

```
Configuration
├── mappedStatements: Map<String, MappedStatement>
│   └── "com.rongx.dao.UserDao.findUser" → MappedStatement
└── resultMaps: Map<String, ResultMap>
    └── "userResultMap" → ResultMap
```

### 1.3 MappedStatement 构建

```java
public static class Builder {
    public MappedStatement build() {
        // 1. 断言检查
        assert configuration != null;
        assert id != null;
        
        // 2. 构建ResultMaps
        mappedStatement.resultMaps = Collections.unmodifiableList(resultMaps);
        
        // 3. 设置主键生成器
        mappedStatement.keyGenerator = isUseGeneratedKeys 
            ? new Jdbc3KeyGenerator() 
            : new NoKeyGenerator();
        
        return mappedStatement;
    }
}
```

## 2. BoundSql - 绑定的SQL

### 2.1 作用

BoundSql存储解析后的SQL语句和参数映射，是Executor实际执行的SQL。

```java
public class BoundSql {
    private String sql;                           // 解析后的SQL (带?)
    private List<ParameterMapping> parameterMappings; // 参数映射
    private Object parameterObject;             // 原始参数对象
    private Map<String, Object> additionalParameters; // 额外参数
}
```

### 2.2 创建过程

```java
public BoundSql getBoundSql(Object parameterObject) {
    return sqlSource.getBoundSql(parameterObject);
}
```

### 2.3 示例

```
原始SQL: SELECT * FROM user WHERE id = #{id} AND name = #{name}
       ↓ (解析)
BoundSql.sql: SELECT * FROM user WHERE id = ? AND name = ?
BoundSql.parameterMappings: [id → Integer, name → String]
```

## 3. ResultMap - 结果映射

### 3.1 作用

ResultMap定义数据库列到Java属性的映射关系。

```java
public class ResultMap {
    private String id;                      // 标识
    private Class<?> type;                 // 结果类型
    private List<ResultMapping> resultMappings; // 属性映射列表
    private Set<String> mappedColumns;      // 已映射的列
}
```

### 3.2 XML配置

```xml
<resultMap id="userMap" type="com.rongx.po.User">
    <id column="id" property="id"/>
    <result column="user_name" property="userName"/>
    <result column="user_age" property="userAge"/>
</resultMap>
```

### 3.3 自动创建

如果未配置resultMap，框架会自动创建基于resultType的映射：

```java
// 基于属性名自动映射
<select id="findUser" resultType="com.rongx.po.User">
    SELECT id, user_name, user_age FROM user
</select>
       ↓ 自动生成
ResultMap:
  id → id
  user_name → userName
  user_age → userAge
```

## 4. ParameterMapping - 参数映射

### 4.1 作用

ParameterMapping定义参数到列的映射，用于设置参数到PreparedStatement。

```java
public class ParameterMapping {
    private String property;       // 参数属性名
    private Class<?> javaType;   // Java类型
    private JdbcType jdbcType; // JDBC类型
    private TypeHandler<?> typeHandler; // 类型处理器
}
```

### 4.2 创建过程

```java
// 从 #{paramName, jdbcType=VARCHAR} 创建
ParameterMapping mapping = new ParameterMapping.Builder()
    .property("paramName")
    .javaType(String.class)
    .jdbcType(JdbcType.VARCHAR)
    .typeHandler(new StringTypeHandler())
    .build();
```

## 6. 构建器层

### 6.1 XMLMapperBuilder

```java
public class XMLMapperBuilder extends BaseBuilder {
    private MapperBuilderAssistant builderAssistant;
    
    public void parse() {
        if (!configuration.isResourceLoaded(resource)) {
            // 解析XML元素
            configurationElement(element);
            // 标记已加载
            configuration.addLoadedResource(resource);
            // 绑定Mapper接口
            configuration.addMapper(Resources.classForName(namespace));
        }
    }
    
    private void configurationElement(Element element) {
        // 1. 解析namespace
        builderAssistant.setCurrentNamespace(namespace);
        
        // 2. 解析resultMap
        resultMapElements(element.elements("resultMap"));
        
        // 3. 解析SQL语句
        buildStatementFromContext(
            element.elements("select"),
            element.elements("insert"),
            element.elements("update"),
            element.elements("delete")
        );
    }
}
```

### 6.2 resultMap解析

```java
private ResultMap resultMapElement(Element node, List<ResultMapping> additional) {
    String id = node.attributeValue("id");
    String type = node.attributeValue("type");
    Class<?> typeClass = resolveClass(type);
    
    List<ResultMapping> resultMappings = new ArrayList<>();
    resultMappings.addAll(additional);
    
    // 解析每个<result>子元素
    for (Element child : node.elements()) {
        resultMappings.add(buildResultMappingFromContext(child, typeClass, flags));
    }
    
    return resultMapResolver.resolve();
}
```

### 6.3 MapperBuilderAssistant

```java
public class MapperBuilderAssistant extends BaseBuilder {
    private String currentNamespace;
    
    public MappedStatement addMappedStatement(...) {
        // 1. 添加namespace前缀
        id = applyCurrentNamespace(id, false);
        
        // 2. 创建Builder
        MappedStatement.Builder builder = new MappedStatement.Builder(
            configuration, id, sqlCommandType, sqlSource, resultType);
        
        // 3. 设置resultMap
        setStatementResultMap(resultMap, resultType, builder);
        
        // 4. 添加到Configuration
        configuration.addMappedStatement(statement);
        
        return statement;
    }
    
    private void setStatementResultMap(...) {
        List<ResultMap> resultMaps = new ArrayList<>();
        
        if (resultMap != null) {
            // 使用指定的resultMap
            resultMaps.add(configuration.getResultMap(resultMapName));
        } else if (resultType != null) {
            // 自动创建Inline ResultMap
            ResultMap inlineMap = new ResultMap.Builder(
                configuration, id + "-Inline", resultType, new ArrayList<>()).build();
            resultMaps.add(inlineMap);
        }
        builder.resultMaps(resultMaps);
    }
}
```

### 6.4 XMLStatementBuilder

```java
public class XMLStatementBuilder extends BaseBuilder {
    public void parseStatementNode() {
        // 1. 获取SQL ID
        String id = element.attributeValue("id");
        
        // 2. 获取SQL类型
        String nodeName = element.getName();
        SqlCommandType commandType = getSqlCommandType(nodeName);
        
        // 3. 获取parameterType
        String parameterType = element.attributeValue("parameterType");
        
        // 4. 获取resultType
        String resultType = element.attributeValue("resultType");
        
        // 5. 创建SqlSource
        SqlSource sqlSource = createSqlSourceFromContext(element, parameterType);
        
        // 6. 创建MappedStatement
        builderAssistant.addMappedStatement(id, sqlSource, commandType, ...);
    }
}
```

## 7. 配置解析流程

### 7.1 整体流程

```
SqlSessionFactoryBuilder.build(reader)
       ↓
XMLConfigBuilder.parse()
       ├─→ parseEnvironments()     解析环境配置
       ├─→ parseMappers()          解析Mapper
       ↓
Configuration (完成)
```

### 7.2 Mapper XML解析流程

```
XMLMapperBuilder.parse()
       ↓
configurationElement(element)
       ├─→ setNamespace()
       ├─→ resultMapElements()
       │        ↓
       │    ResultMap → Configuration.resultMaps
       │
       └─→ buildStatementFromContext()
                ↓
            XMLStatementBuilder.parseStatementNode()
                ├─→ createSqlSource()
                ├─→ createParameterMapping()
                └─→ addMappedStatement()
                         ↓
                     MappedStatement → Configuration.mappedStatements
```

## 11. 面试问题

### Q1: MappedStatement的作用？
**Answer**: 存储SQL配置，包括SQL文本、参数映射、结果映射、主键生成器等。

### Q2: resultMap和resultType的区别？
**Answer**: resultMap需要显式配置列到属性的映射；resultType自动根据属性名映射。

### Q3: #{}和${}在解析阶段的区别？
**Answer**: #{}生成?占位符，${}直接字符串拼接。

## 12. 总结

映射层要点：
1. MappedStatement是SQL配置的核心
2. Builder负责解析XML/注解生成配置
3. ResultMap定义结果映射
4. BoundSql是Executor实际执行的SQL
